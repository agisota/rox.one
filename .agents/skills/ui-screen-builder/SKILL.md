# UI Screen Builder

For UI work:
1. Find existing component patterns.
2. Add component tests before implementation.
3. Test render, labels/text, keyboard access, click behavior, state persistence, and empty/error/loading states.
4. Add E2E smoke test for user-visible flows when the repo has a suitable harness.
5. Avoid visual clutter.
