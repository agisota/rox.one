# T194 - CredentialRequest Invalid Field Visual Polish

## 1. Task summary

Apply Tailwind `aria-invalid:*` variant classes to all 4 inputs in `CredentialRequest` to provide visual-level reinforcement of the invalid state already wired by T185. Pure className additions: `aria-invalid:border-destructive aria-invalid:bg-destructive/5 aria-invalid:focus-visible:ring-destructive/20`. No JSX logic changes.

## 2. Repo context discovered

- T185 wired `aria-invalid={hasError}` onto each input and added error `<div role="status" className="text-destructive">` messages. The `aria-invalid` attribute is present in the DOM when a field fails validation; this ticket makes the field itself visually communicate the error state.
- Tailwind's `aria-invalid:*` variant is a built-in feature (available since Tailwind v3.3). It applies styles when the element's `aria-invalid` attribute is `true` or `"true"`. No custom plugin or arbitrary selector needed.
- `border-destructive`: maps to `hsl(var(--destructive))` — a red/orange tone in the design system's light and dark themes. Provides immediate visual cue.
- `bg-destructive/5`: a 5% opacity destructive background tint. Subtle enough not to overwhelm the field content, visible enough to connect the field to the error state.
- `focus-visible:ring-destructive/20`: changes the focus ring color when the invalid field is focused. Without this, the field would show the default `ring-ring` focus ring, which is tonally unrelated to the error state and confusing for keyboard users.
- The 4 inputs are: `username`, `password`, `host`, `apiKey` (or equivalent field names — confirmed by reading the component).

## 10. Remaining risks

- **Pure visual change; visual regression possible at edge zoom levels.** At 200% zoom, the `bg-destructive/5` tint may combine with the browser's own high-contrast mode and produce an unexpected color. This is a low-probability edge case, acceptable for the polish scope.
- **`aria-invalid:focus-visible:ring-destructive/20` applies only on keyboard focus.** Mouse-focused invalid fields will show the border and background tint but not the ring (because `focus-visible` only activates for keyboard navigation). This is correct behavior per WCAG — ring is a keyboard focus indicator and should not appear on mouse click.
