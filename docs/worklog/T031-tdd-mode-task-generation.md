# T031 TDD Mode Task Generation

## 1. Task summary

Add a shared TDD task-pack generator for Agent Workbench that turns a scoped request and validation gates into deterministic red/green/verify/worklog tasks.

## 2. Repo context discovered

- The ticket file `docs/tickets/T031-tdd-mode-task-generation.md` is a placeholder and points back to the master backlog.
- `packages/shared/src/workbench/spec-compiler.ts` already emits basic implementation tasks when `tdd:test-first` is selected.
- `packages/shared/src/workbench/agent-pipeline-planner.ts` already models TDD/testing lanes for multi-agent plans.
- There is no standalone TDD task-pack generator that can be reused by TDD mode UI/runtime flows.
- `packages/shared/src/workbench` is the existing pure-policy surface for workbench modules and Bun unit tests.
- Validation gates already cover unit/integration/UI/E2E/security/RBAC/quota/sync checks.

## 6. Implementation changes

- Added `packages/shared/src/workbench/tdd-task-generator.ts` as a pure shared generator.
- Added Zod schemas for generated TDD task packs, task phases, and generator input.
- Generated deterministic task IDs for red, green, verify, and worklog phases from the ticket ID.
- Mapped validation gates to concrete placeholder commands so downstream UI/runtime flows can show executable checks.
- Added fake-provider requirements for browser/research, auth/session, team/RBAC, storage/S3/quota, and sync/cloud surfaces.
- Added markdown rendering for checklist-style TDD plans without claiming completion.
- Exported the generator from `packages/shared/src/workbench/index.ts`.
