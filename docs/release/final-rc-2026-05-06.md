## 1. Executive Summary

ROX ONE is now a coherent private release candidate on top of Craft Agents OSS
v0.9.1. The T074-T097 wave turns the Experience Layer from a set of attractive
screen previews into a replayable runtime loop with typed events, reducer truth,
deterministic fake providers, mission evidence, validation gates, metrics,
quest progression, account/share contracts, security checks, and release
evidence. The T098-T122 continuation adds screenshot-backed Experience tab
proof, Electron smoke/startup/package hardening, ROX.ONE active backend copy,
runtime-artifact git hygiene, public-risk boundaries, build-warning gates,
fresh bundle evidence, release-evidence reconciliation, and a Mac
private-release trust-boundary gate plus packaged artifact validation before
Mac ARM workflow upload without claiming public-production readiness.

This RC is production-ready for private validation and operator review. It is
not yet a public SaaS launch because real external providers, hosted
persistence, public viewer infrastructure, signed distribution, and production
secrets are intentionally outside the fake-provider-safe test boundary.

```text
User action
  -> ExperienceEvent
  -> ExperienceRuntimeStore reducer
  -> persistence adapter seam
  -> fake provider or provider adapter seam
  -> artifact/evidence
  -> validation gate
  -> metric update
  -> quest/progression update
  -> ledger/audit event
  -> UI feedback state
```

## 3. Product Architecture

```text
Electron Renderer
  -> Composer / Prompt Lab / Spec Builder / TDD Plan / Review Gate
  -> Deep Missions / Mission Control / Arena / Progress / Quest Map / Forge
  -> ROX ID / Account / Share surfaces
  -> ExperienceGlobalHud

Preload and IPC
  -> typed command DTOs
  -> safe renderer-visible state

server-core
  -> sessions and share provider
  -> mission scheduler contracts
  -> provider gateway contracts
  -> persistence contracts

packages/shared
  -> ExperienceRuntimeStore
  -> mission mode prompt registry
  -> event replay, selectors, metrics, quests, integrity rules
```

## 4. Account Sequence

```text
Open ROX ID
  -> choose login/register/reset
  -> renderer validates form
  -> account command over IPC
  -> main process account/session boundary
  -> confirmed session required for authenticated state
  -> encrypted local session where available
  -> account summary hydrates
  -> logout clears local and session state
```

Registration pending is not presented as authenticated success. Raw IPC errors
are classified into safe Russian feedback states.

## 5. Mission Launch Sequence

```text
Deep Missions form
  -> validate title/objective/input/budget/tokens/storage/agents/VDI target
  -> mission.drafted
  -> persist draft
  -> launch through scheduler seam
  -> mission.launched
  -> Mission Control projection
  -> checkpoint action
  -> artifact.created
  -> gate.passed | gate.warned | gate.failed
  -> mission.finalized only when final artifact and passing gate exist
```

Elapsed time alone never completes a mission.

## 6. Experience Event Sequence

```text
prompt.submitted
  -> prompt.rewritten
  -> spec.compiled
  -> tdd.plan.created
  -> review.completed
  -> gate.warned or gate.passed
  -> mission.drafted
  -> mission.launched
  -> mission.checkpoint.completed
  -> artifact.created
  -> quest.completed
  -> reward.unlocked
  -> ledger.entry.recorded
```

Command, Game, and Arena modes read the same truth. Presentation mode cannot
complete gates or inflate VDI.

## 7. Provider Gateway Sequence

```text
Mission draft
  -> MissionModePromptRegistry.compile()
  -> provider request with contract and required artifacts
  -> deterministic fake provider in tests
  -> real adapter seam in production wiring
  -> output schema validation
  -> malformed output fails gate
  -> timeout/retry/error taxonomy
  -> secret redaction before public/share artifacts
```

Tests do not call real LLM, browser, S3, email, payment, marketplace,
shortlink, or public viewer providers.

## 8. Share Sequence

```text
Share button
  -> build public-safe bundle
  -> redact secrets, cookies, API keys, private file content
  -> ShareProvider.uploadBundle()
  -> ShareProvider.createShortlink()
  -> ShareProvider.getShareStatus()
  -> copied | failed_retryable | failed_permanent | revoked
```

The RC does not fake a public URL as production truth. Public share remains
provider-backed through the contract.

## 9. Quest And Progression Sequence

```text
Verified artifact
  -> passing validation gate
  -> metric engine
  -> quest dependency graph
  -> reward/unlock event
  -> ledger/audit event
  -> HUD and Progress/Quest Map projection
```

Paid entitlement may change capacity metrics only. It cannot satisfy evidence,
VDI, quality, readiness, leaderboard, or validation gates.

  but bundle-size warnings remain observational only; Electron/WebUI/Viewer
  still need a separate bundle-splitting policy before current large chunks
  become a CI gate.
- Public production remains blocked by real provider orchestration, hosted
  persistence/workers, public share storage/shortlinks, ROX ID email
  verification, payments, signed/notarized distribution, observability,
  dependency audit remediation or signed accepted-risk approval, and external
  security review.
- T121 prevents the private Mac ARM workflow from uploading artifacts without
  first validating the current local-RC trust boundary, but it does not create a
  Developer ID signature or notarized production artifact.
- Electron smoke is green on a GUI-capable non-sandbox launch surface; the
  sandboxed command wrapper still aborts before app markers and is tracked as
  an environment constraint rather than a repo startup failure.
