## 3. System Shape

```text
User
  -> Electron Renderer
      -> Composer / Prompt Lab / Spec Builder / Review Gate
      -> Experience Layer screens
      -> ROX ID account cabinet
  -> Preload bridge / Electron main
  -> server-core handlers and domain services
  -> persistence adapter contracts
  -> provider gateway contracts
  -> artifact / evidence / audit records
  -> validation gates
  -> metrics projectors
  -> renderer selectors and user feedback states
```

Important boundary:

```text
Command mode
Game mode
Arena mode
  -> all render one shared mission truth
  -> none can buy or click evidence into existence
  -> none can mutate gates differently
```

## 4. Mission Runtime Flow

```text
Create mission
  -> validate preset, budget, selected agents, checkpoint cadence
  -> persist MissionRun
  -> persist MissionCheckpoint records
  -> enqueue deterministic/future scheduler jobs
  -> execute due checkpoint
  -> write artifacts and gate evidence
  -> project VDI / Quality / Execution Readiness
  -> block final completion unless evidence exists
```

State model:

```text
draft
  -> queued
  -> running
  -> blocked | completed | failed | cancelled

checkpoint:
  queued
  -> running
  -> completed | failed | blocked
```

Hard rules:

- elapsed time alone does not complete a mission;
- final pass requires artifact/gate evidence;
- negative or invalid cost events cannot inflate budget;
- expensive swarm expansion requires explicit approval;
- paid entitlement can increase capacity, not quality.

## 6. Account And Registration Flow

ROX ID is embedded inside the app settings page. The screen now treats
registration as a pending-verification state, not as a successful authenticated
session.

```text
User enters display name/email/password
  -> Renderer validates form intent
  -> account:request IPC call
  -> Electron main account proxy
  -> account API contract
  -> register response
  -> pending verification state in UI
  -> user verifies email externally
  -> user signs in
  -> rox_session captured by main process
  -> Electron safeStorage persists encrypted session
  -> account cabinet hydrates on restart
```

Security boundary:

```text
rox_session
  -> main process only
  -> encrypted local persistence
  -> never exposed to renderer state
  -> never included in public share payloads
```

## 7. Share Flow

```text
Share button
  -> build public-safe session bundle
  -> sanitize recursive payload
  -> ShareProvider.uploadBundle()
  -> ShareProvider.createShortlink()
  -> validate public URL shape
  -> show copied/failed/auth-required UI state
```

The RC includes a provider seam and fake-provider tests. It does not yet include
a production public viewer/shortlink service.
