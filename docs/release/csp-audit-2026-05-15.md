# CSP Audit - 2026-05-15

Auditor: T509 automated audit pass
Branch: `chore/T509-csp-cve-renovate`
Prior audit reference: T249 (`feat/T249-csp-zod-boundary-hardening-v2`)

## Scope

All four Electron renderer HTML files that carry a `Content-Security-Policy` meta tag. The
Electron main process has no HTTP response-header CSP override; security relies entirely
on these meta tags enforced by Chromium's renderer-process CSP enforcement.

## Directive-by-Directive Assessment

### 1. `default-src 'self'`

| | |
|---|---|
| Threat covered | Catch-all for any resource type not explicitly overridden. Blocks external origins as fallback. |
| Status | **PASS** — all four files set `default-src 'self'`. |

### 2. `script-src`

| Source value | Status | Notes |
|---|---|---|
| `'self'` | PASS | Bundles from same origin only. |
| `'unsafe-eval'` | **PASS** | Absent in all four files. T249 correctly removed this. No regression. |
| `'wasm-unsafe-eval'` | PASS (conditional) | Required for WASM inference binary. Distinct from `unsafe-eval` for JS; acceptable. |
| `'unsafe-inline'` | **PASS** | Absent from `script-src` in all four files. |
| `http://localhost:8097` | CONCERN (low, dev-only) | React DevTools server present in `index.html` and `playground.html`. Should be gated behind a build flag in production. Tracked in T-T249-FOLLOWUP-csp-unsafe-inline-script.md. |

**Overall script-src: PASS** — no unsafe-eval, no unsafe-inline. Localhost:8097 dev endpoint
present in production HTML is low-risk but should be addressed in v1.1.0.

### 3. `style-src`
