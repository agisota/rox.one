## P0 BLOCKERS surfaced by 2026-05-20 PR #268 audit

The audit verdict for PR #268 was **conditional-go**; the merge of `f4fd8ac3`
left three packaging-side P0 blockers open. RC8 MUST NOT cut until all three
are closed.

### B-CI-1 — signed-release workflows bypass `rox-design:payload:verify`

The gate I added in PR #268 lives inside `package.json` `electron:dist*`
entries. Several release workflows invoke `bunx electron-builder` directly,
bypassing the bun-script chain:

| File | Line | Invocation |
|---|---|---|
| `.github/workflows/mac-signed-release.yml` | 144 | `bunx electron-builder --mac --arm64 --publish=never` |
| `.github/workflows/linux-signed-release.yml` | 146 | `bunx electron-builder --linux --publish=never` |
| `.github/workflows/windows-unsigned-release.yml` | 136 | `bunx electron-builder --win --x64 --publish=never` |
| `.github/workflows/release-all-platforms.yml` | 269 | `bunx electron-builder --linux --publish=never` |
| `.github/workflows/release-all-platforms.yml` | 302 | `bunx electron-builder --win --x64 --publish=never` |

Only `.github/workflows/release-all-platforms.yml:255` (mac arm64 lane)
goes through `bun run electron:dist:dev:mac:arm64` and inherits the gate.

**Fix**: hoist the payload verify into `apps/electron/scripts/beforeBuild.cjs`
(the electron-builder `beforeBuild` hook). This is invoked by electron-builder
regardless of entry point, so every signed/unsigned/CI/local invocation is
gated identically. The existing hook only returns `false` to skip auto
dependency collection — easy to extend with a `execFileSync('bun', ['run',
verifier])` step gated by `ROX_SKIP_ROX_DESIGN_PAYLOAD_VERIFY=1` for parity
with the bun-script path.

Defence-in-depth: also prepend `bun run rox-design:payload:verify` as an
explicit step in the 5 workflow files above, so CI surfaces the gate as a
labelled job instead of failing inside the electron-builder log.

### B-REPRO-2 — non-reproducible payload preparation

`scripts/prepare-rox-design-runtime.ts:5` reads from host-local
`/Applications/Open Design.app/Contents/Resources` by default with no
SHA-256 pinning. Two issues:

- Different mac hosts may have different Open Design 0.7.0 builds with
  identical version strings; the resulting payload diverges silently.
- CI lanes without `/Applications/Open Design.app/` (Linux runners,
  fresh mac runners) cannot prepare a payload without manual seeding.

**Fix**: add `--from-archive=<url|tarball-path>` mode that fetches a
SHA-256-pinned tarball (uploaded by a single canonical mac build job).
Track in Goal #6 of the 2026-05-20 top-10 list.

### B-SIGN-3 — nested Mach-O `node` binary lacks `mac.binaries`/`signIgnore`

`apps/electron/resources/rox-design/open-design/bin/node` is a nested
Mach-O binary in the payload. `electron-builder.yml` `mac` config does not
list it under `binaries:` for re-signing, nor does it list it under
`signIgnore`. Apple notarization will reject the bundle when the payload is
populated and signed.

**Fix**: add to `electron-builder.yml`:
```yaml
mac:
  binaries:
    - resources/rox-design/open-design/bin/node
```
Or, if the binary is already correctly signed by Open Design's release
process, add to `signIgnore` and document the upstream signature provenance.

4. `bun test` (full unit suite — bun-test green)
5. `bun run validate:linux-deb-rpm` (PR #265 deliverable)
6. `bun run validate:linux-signed-release-pipeline`
7. `bun run validate:windows-private-release-boundary` +
   `validate:windows-boundary-fixtures`
8. `bun run validate:mac-private-release-boundary` +
   `validate:mac-boundary-fixtures`
9. Packaged-mac smoke on Sonoma host: `electron:dist:dev:mac:arm64` →
   `electron:smoke:packaged:mac` → `electron:ui-smoke:packaged:mac` →
   `validate:packaged-artifacts`. **Requires Open Design 0.7.0 resources or
   mock payload (`rox-design:prepare --force`) before electron-builder runs.**
10. `bun run rc:preflight` (existing RC preflight gate).
11. `bun run rox-design:payload:verify` against real or mocked payload — added
    in PR #268 follow-up `bfd038f6`. Already wired into every `electron:dist*`
    entry, so passes implicitly if gate 9 passes.
