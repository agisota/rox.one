# v1.0.0-rc.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tag `v1.0.0-rc.2` with unsigned Mac+Windows beta artifacts, GPG-signed Linux artifacts, hardened test infrastructure, and security hygiene cleanup; restart 72h soak from new tag.

**Architecture:** Single orchestrator session dispatches 12 isolated agents in a batch, each owning a disjoint file scope. Tiered merge ordering (orthogonal → workflows → consumers) with auto-revert on preflight regression. Post-merge: cut tag, trigger 3 parallel CI workflows, download artifacts, verify 16/16 preflight, start soak.

**Tech Stack:**
- `gh` CLI for GitHub Actions, PRs, secrets, releases
- `bun` test runner + Vite/Rollup for renderer build
- `electron-builder` for platform packaging
- `gpg` for Linux GPG signing
- `@cyclonedx/cdxgen` for SBOM
- `gitleaks` for secret scanning
- `husky` for pre-commit
- Conventional Commits for all PRs

**Spec:** [`docs/superpowers/specs/2026-05-15-v1.0.0-rc.2-design.md`](../specs/2026-05-15-v1.0.0-rc.2-design.md)

## Phase 3 — PR aggregation + mergeability check

### Task 3.1: Check each PR

**Files:** None

- [ ] **Step 1: Inspect each PR**

For each PR# in `/tmp/rc2-pr-map.txt`:
```bash
for ENTRY in $(cat /tmp/rc2-pr-map.txt); do
  ID=${ENTRY%%=*}
  PR=${ENTRY##*=}
  [ "$PR" = "NONE" ] || [ "$PR" = "BLOCKED" ] && continue
  STATUS=$(gh pr view $PR --json mergeable,mergeStateStatus,statusCheckRollup --jq \
    '"mergeable=" + .mergeable + " state=" + .mergeStateStatus + " checks=" + ([.statusCheckRollup[]? | .conclusion // "PENDING"] | join(","))')
  echo "$ID PR #$PR $STATUS"
done | tee /tmp/rc2-pr-status.txt
```

Expected: each PR shows `mergeable=MERGEABLE` and `state` is one of `CLEAN`, `UNSTABLE` (pre-existing CI failures OK; new ones not), or `BEHIND` (needs rebase).

- [ ] **Step 2: Resolve conflicts and CI failures**

For each PR with `mergeable=CONFLICTING` or `state=DIRTY`:
```bash
gh pr update-branch $PR --rebase 2>&1 | head -3
```

If still conflicting after rebase, spawn a conflict-resolver agent (Appendix B template).

For each PR with new CI failures (not pre-existing): spawn a fix agent (Appendix C template) with the failure log.
