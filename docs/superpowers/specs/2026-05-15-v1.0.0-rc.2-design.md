## Problem statement

`v1.0.0-rc.1` was tagged on 2026-05-14 at SHA `0b0a218f` with the bundle-budget
gate green via PRs #207/#208/#216. The 72-hour soak window opened at
2026-05-14T10:02:42Z.

During the soak, two facts surfaced:

1. **Signing secrets are absent across all platforms.** `gh secret list` returns
   empty. The repo has three release workflows (`mac-signed-release.yml`,
   `linux-signed-release.yml`, `private-release.yml`) and zero matching
   Apple/Linux GPG secrets. The `packaged-artifacts` preflight gate cannot pass
   without signed artifacts being downloaded into `apps/electron/release/`.

2. **No Windows release workflow exists at all.** The
   `v1-github-release-template.md` lists Windows as a target artifact, but
   there is no `.github/workflows/windows-*-release.yml`.

The operator has decided:

- No Apple Developer membership ($99/yr) for this release cycle — Mac ships
  unsigned beta.
- No Windows code-signing certificate ($200-500/yr) — Windows ships unsigned
  beta.
- Linux ships GPG-signed (GPG keys generated and configured during this RC2
  cycle; free).
- Cut a new RC tag (`v1.0.0-rc.2`) with the additional cleanup work, restart
  the 72-hour soak. The current `v1.0.0-rc.1` tag will remain in history but
  is superseded.

Additionally, the operator chose to bundle three workstreams into this RC:

- **Track A (release shipping):** unsigned Mac + Windows workflows, GPG keys
  + Linux signing, validator updates.
- **Track C (test infrastructure):** ModalProvider RTL helper, bun:test vs
  vitest drift investigation, vitest config consolidation.
- **Track D (security hygiene):** gitleaks pre-commit hook, CycloneDX SBOM in
  release pipelines, CSP+CVE audit refresh + Renovate enable, RPC rate-limit
  audit.

Total: 12 independent work units.

## Out of scope

Explicitly **not** in this spec:

- **Apple/Windows code-signing** — requires paid certificates the operator has
  not procured. Deferred to v1.0.x or v1.1.0.
- **R.11 destructive history rewrite** — gated on v1.0.0 GA tag, separate spec.
  Merging this RC2 spec/plan intentionally preempts R.11 until the RC2 lane is
  either merged or abandoned; after this PR lands, run a report-only R.11
  blocker refresh before any destructive R.11 action.
- **Major-version upgrades** (TS6 / Vite8 / React19 / Electron42) — separate
  v1.1.0 roadmap.
- **Performance follow-ups** beyond what bundle-budget already enforces.
- **Net-new features** — every unit is hardening, infrastructure, or
  cleanup.
- **Production code rate-limit changes** — D4 produces only an audit document;
  implementation tickets land in v1.0.x.

## Success criteria

The spec is **DONE** when:

1. All 12 work units have either landed on `main` or carry an explicit
   `Status: DEFERRED` ticket with v1.0.x or v1.1.0 owner.
2. `bun run rc:preflight` returns 16/16 green on `main` HEAD with signed and
   unsigned artifacts present in `apps/electron/release/`.
3. Tag `v1.0.0-rc.2` is pushed to `origin`, annotated with the evidence
   packet template from `v1-rc-tag-decision-matrix.md`, with all
   `<SOAK_PASSING_SHA>` and timestamp placeholders filled.
4. GitHub Release `v1.0.0-rc.2` exists as a prerelease draft with attached
   artifacts (Linux GPG-signed `.deb`/`.rpm`/`.AppImage`, unsigned-beta Mac
   `.dmg`, unsigned-beta Windows `.exe`, `sbom.json` per platform).
5. The 72-hour soak timer has restarted from the new tag push timestamp.
6. Memory has been updated with the RC2 outcome plus any race-condition
   lessons from the swarm execution.

## Architecture

### 1. Topology

A single orchestrator (this Claude session, optionally a follow-on session if
context fills) dispatches 12 agents in one `Agent[]` batch with
`isolation: "worktree"`. Each agent:

- Receives a self-contained prompt (no inherited session context).
- Operates in `.claude/worktrees/agent-<id>/` (auto-cleanup on no changes,
  preserved on changes).
- Creates exactly one branch off `origin/main` (latest at dispatch time).
- Produces exactly one PR via `gh pr create`.
- Returns a structured summary: PR number, files changed, validators run,
  any deviations.

The orchestrator stays in the parent worktree at `/home/dev/craft/rox-one-terminal`
and consumes task notifications to track completion. Aggregation, merge order,
preflight gating, and tag cut happen serially in the orchestrator after all
agents complete.

### 2. Coordination

**Ralph pause.** Before spawn, the orchestrator exports `DISABLE_OMC=1` and
`OMC_SKIP_HOOKS=ralph,autopilot,ultrawork` to its session env. Any existing
ralph/autopilot processes are checked via `ps -ef | grep -iE 'ralph|autopilot'`
and surfaced to the user if active. The spec assumes the user will not start
ralph in a parallel session for the duration of RC2.

**Worktree isolation.** No two agents share a filesystem state. Conflicts can
only manifest at merge time, never during write time.

**Branch naming.** Conventional prefixes (`feat/` `fix/` `chore/` `docs/`)
are enforced in each agent prompt. Each prompt lists the agent's exact
branch name to prevent collisions.

**Negative constraints.** Each agent prompt contains an explicit deny list of
files owned by other agents. This is the primary defense against
file-overlap bugs at merge time. The deny list is generated from the
authoritative file ownership table in this spec (see Section 3).

### 4. Merge ordering (Tier graph)

Agents work in parallel; merges are serialized by orchestrator after all PRs
arrive.

```
Tier 1 (orthogonal — squash-merge in any order):
  A1, C1, C3, D1, D3, D4    (6 PRs)
  C2 if successful, else mark BLOCKED + skip

Tier 2 (workflows — must precede their consumers):
  A2 (mac-unsigned-release.yml)
  A3 (windows-unsigned-release.yml)

Tier 3 (depends on Tier 2):
  A4 (packaged-artifacts validator — needs new workflows to exist)
  D2 (SBOM step — needs all 3 workflows to exist)
```

Tier 1 → preflight after each merge. Tier 2 → batch merge then preflight.
Tier 3 → rebase against latest main first, then merge, then preflight.

### 5. Data flow per agent


### 6. Prompt template

Every agent prompt follows this structure (~600-1000 words):

```
Task: <1-line task>

REPO: /home/dev/craft/rox-one-terminal (isolated worktree)
BASE: origin/main
WORKING BRANCH: create `<prefix>/<slug>` from origin/main
TICKET REF: T-XXX (or new RC2 ticket)

CONTEXT:
  <2-3 paragraphs explaining WHY for RC2, current state,
   what this unblocks>

YOUR TASK:
  1. <concrete step>
  2. <concrete step>
  ...
  N. ship: commit, push, open PR via `gh pr create`

FILES YOU MAY MODIFY (whitelist):
  - path/to/file/1
  - ...

FILES YOU MUST NOT TOUCH (deny list — other agents own them):
  - path/to/file/owned/by/another/agent
  - ... (full enumeration from Section 3)

OPERATIONAL CONSTRAINTS:
  - Do NOT modify production app code unless explicitly in whitelist
  - Do NOT raise budget ceilings, lower coverage, disable validators
  - Do NOT use --no-verify or skip hooks
  - Conventional Commits format
  - PR body: ## Summary / ## Test plan / ## Why this RC2 / cross-link
    docs/superpowers/specs/2026-05-15-v1.0.0-rc.2-design.md

VALIDATION (before declaring done):
  - <task-specific validators that must exit 0>
  - bun run typecheck (if production code touched)
  - bun run lint:<package> (if package touched)

RETURN (concise — under 200 words):
  - PR number + URL
  - Files actually changed (git diff --stat)
  - Validators that ran + exit codes
  - Any deviations from spec + why
```

## Error handling

### Failure taxonomy

| # | Failure type | Detection | Recovery |
|---|--------------|-----------|----------|
| 1 | Agent crashes mid-task | task-notification status≠completed | Spawn replacement agent with same prompt |
| 2 | Agent declares "blocked" | result contains `STATUS: BLOCKED` | Skip if optional (C2); halt + report if required |
| 3 | Agent produces broken PR | `gh pr view` shows CI failure | Spawn fix agent with failure log |
| 4 | Merge conflict | `gh pr merge` HTTP 409 | Auto `gh pr update-branch --rebase`; if still conflict → conflict-resolver agent (opus) |
| 5 | Preflight regression after merge | `bun run rc:preflight` red | Auto `git revert HEAD`; mark PR regression; spawn debug agent |
| 6 | Workflow CI fails post-tag | `gh run view` failure | Per-platform diagnosis, retry / patch + new RC / caveat |
| 7 | Signed-build artifacts mismatch | `validate-packaged-artifacts` fail | Per-platform diagnosis |
| 8 | Ralph autonomous mode interferes | Unexpected branch switches | Halt, surface to user |
| 9 | Spending limit re-locks | Mid-build GH error | Halt, surface |
| 10 | Apple/Win signing scope creep | Agent A2/A3 requests cert | Reject — unsigned only this spec |

### Required vs optional units

**Required** (halt if unresolvable):
- A1, A2, A3, A4 (release infrastructure — without these, no RC2)
- D1, D2 (security baseline — gitleaks + SBOM are user-rule requirements)

**Optional** (mark deferred + continue):
- C2 (test drift — already deferred in memory; treat investigation as bonus).
  If timebox hits 30min without root cause, push investigation doc and
  reopen ticket for v1.0.x.
- C3 — never blocks RC2. Produces a config consolidation, can ship in v1.0.x
  if it surfaces unexpected conflicts with C1's helper at merge time.
- D3 — produces docs + Renovate config; can defer Renovate enable to v1.0.x
  if it generates excessive PR churn during soak.
- D4 — audit document only, no production code change. Always optional;
  output is the input for implementation tickets in v1.0.x.

### Revert hygiene

Only `git revert` on main. Never `reset --hard` or force-push. This matches
`<git_and_versioning>` in user CLAUDE.md.

### Surface to user only on

1. Required unit halt
2. Spending limit re-locks
3. Ralph activity detected
4. Apple/Windows cert request (scope creep)
5. Preflight regression after debug agent fails
6. Soak P0 during 72h window

Everything else handled silently via decision tree.

## Testing strategy

```
PHASE 0  (pre-spawn)        rc:preflight 16/16 baseline
PHASE 1  (per-agent)        agent-local: typecheck, lint, unit-of-task validator
PHASE 2  (per-merge)        rc:preflight after each Tier 1/2/3 merge
PHASE 3  (pre-tag)          full sweep: typecheck:all, lint, test, rc:preflight
PHASE 4  (post-tag CI)      linux-signed + mac-unsigned + windows-unsigned, parallel
PHASE 5  (post-artifact)    packaged-artifacts gate + final rc:preflight 16/16
PHASE 6  (soak, 72h)        observation: zero P0, zero release-blocker
PHASE 7  (M.21 promote)     per existing runbook in /tmp/v1.0.0-promote-runbook.md
```

## Operator constraints

For the orchestrator session to succeed, the operator must:

1. **Not start ralph/autopilot/ultrawork** in any other Claude session on this
   repo for the spec duration.
2. **Keep GitHub Actions spending limit ≥ $10** (or unlimited). The 3
   parallel release workflows will consume Mac/Win minutes (10× / 2×
   multipliers).
3. **Approve admin merges** at orchestrator request points (the design
   assumes `gh pr merge --admin` rights — required because branch protection
   is currently absent but if enabled later, admin override needed).
4. **Acknowledge unsigned-beta release notes** — Mac/Windows artifacts will
   ship with platform warnings and explicit "unsigned beta" disclosure.
