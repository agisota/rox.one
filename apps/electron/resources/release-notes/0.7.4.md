# v0.7.4 — Кастомные эндпоинты и переработка ветвления сессий

---

## Возможности

- **Поддержка кастомных эндпоинтов** — подключение любого OpenAI-совместимого провайдера через `registerProvider` Pi SDK с селектором протокола, полем ввода base URL и регистрацией моделей.

## Улучшения

- **UX кастомных эндпоинтов** — селектор протокола теперь отображается перед полем ввода base URL, а персистентность состояния редактирования работает надёжно при маршрутизации настройки.
- **Очистка CustomEndpointApi** — дедуплицировано определение типа, извлечён `InitMessage` и исправлен баг замены при регистрации моделей.

## Исправления

- **Переработка ветвления сессий** — полностью переработан поток branch/fork для устранения ошибок preflight, несоответствий CWD и зависших сессий. Ветви теперь создаются при первом сообщении пользователя вместо preflight, разговор корректно обрезается в точке ветвления, а имена `spawn_session` корректно отображаются в боковой панели.
- **Надёжность отсечения ветвей** — отсечения ветвей Pi и Claude теперь используют якоря ходов sidecar и охранные механизмы lineage с корректным резервным вариантом при отсутствии UUID.
- **Windows: дублирование сообщений при ветвлении** — `fs.watch()` в Windows агрессивно срабатывает при атомарных записях, что вызывало повторную трансляцию метаданных сессии ConfigWatcher-ом и дублирование сообщений. Исправлено через дедупликацию по ID в `appendMessage()`, подавление собственных событий через сравнение сигнатур и увеличение debounce до 300ms на Windows.
- **Windows: пустые сообщения ветвления** — сообщения ветвления сохранялись с захардкоженными путями исходной сессии вместо портативных токенов `{{SESSION_PATH}}`, что вызывало ошибки разрешения путей на Windows. Исправлено через повторную токенизацию путей из исходной директории в директорию ветви при копировании сообщений.
- **Переключение моделей в середине сессии** — `ClaudeAgent.setModel` теперь корректно вызывает `super.setModel` для обновления внутренней ссылки на модель, устраняя случаи, когда изменения модели не вступали в силу.
- **Крах при клике на ссылку JSON-файла** — клик по ссылкам JSON-файлов для несуществующих файлов больше не вызывает крах приложения.
- **`cache_control` в пустых текстовых блоках** — удалён `cache_control` из пустых текстовых блоках в запросах к Anthropic API, предотвращающий ошибки валидации.
- **Таймаут отправки автоматизаций** — увеличен send-timeout для рабочего процесса ежедневных точек обсуждения для предотвращения преждевременных таймаутов.
