# v0.8.3 — Самоуправление сессий, компактный режим и надёжность автоматизаций

## Возможности
- **Инструменты самоуправления сессий** — Агенты теперь могут устанавливать собственные метки, статус и запрашивать другие сессии через `set_session_labels`, `set_session_status`, `get_session_info` и `list_sessions`. Позволяет создавать автоматизированные рабочие процессы самозакрытия, где агент завершает работу и сигнализирует об окончании.
- **UI компактного режима** — Добавлены элементы управления компактного режима для режима разрешений, вложений, источников и выбора рабочей директории с drawer-селекторами, а также компактные макеты чата и информации о сессии.

## Улучшения
- **Очистка устаревших провайдеров** — Удалены устаревшие типы провайдеров и улучшена обработка ошибок при настройке подключений.
- **Производительность сборки Docker** — Замена тотального `chmod -R` на целевые команды `find`, значительно ускоряющая сборку Docker-образов.
- **Мультиархитектурная CI для Docker** — Release и promote-workflow теперь собирают мультиархитектурные Docker-образы.

## Исправления
- **fs.watch откатывает метаданные idle-сессий** — Исправлено состояние гонки, при котором файловые наблюдатели откатывали программные записи метаданных (метки, статус) в idle-сессиях.
- **Устаревший lock-файл сервера** — Обработка устаревших файлов `.server.lock` при перезапусках Docker-контейнеров для чистого запуска сервера.
- **Автоматизации не срабатывают на headless-серверах** — Нетерпеливая инициализация системы автоматизаций при загрузке вместо ленивой, так что запланированные и событийно-управляемые автоматизации работают с первого запуска.
- **Тихие сбои setSessionStatus** — Корректное ожидание асинхронного вызова `setSessionStatus` для отображения ошибок вместо их тихого поглощения.
- **Инструменты сессий для других сессий** — `set_session_labels` и `set_session_status` теперь могут корректно работать с сессиями, отличными от вызывающей.
- **Валидация инструментов сессий** — ID меток и ID статуса теперь валидируются перед применением, предотвращая загадочные ошибки от невалидных значений.
- **Отображаемые имена инструментов сессий** — Инструменты управления сессиями теперь показывают человекочитаемые имена в UI TurnCard.
- **Обновление моделей при настройке подключения** — Ожидание обновления списка моделей после настройки нового подключения, предотвращая устаревшие списки моделей.
- **Игнорирование предпочтения соавтора** — Учитывается настройка пользователя `includeCoAuthoredBy: false` для git commit-трейлеров.
- **Маршрутизация локальных файловых URL** — Исправлена маршрутизация markdown для URL `file://`, так что ссылки на локальные файлы отображаются корректно.
- **Обрезка текста в уведомлениях** — Текст описаний в toast-уведомлениях может переноситься до 2 строк вместо обрезки.
- **Тег образа Docker Compose** — Обновлён образ сервера по умолчанию до v0.8.2 в конфигурации Docker Compose.

## Критические изменения
