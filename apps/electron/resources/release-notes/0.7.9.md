# v0.7.9 — Надёжная доставка WebSocket, переработка моделей Copilot и исправления Windows

---

## Возможности

- **Надёжная доставка событий WebSocket** — Соединения WebSocket теперь используют отслеживание порядковых номеров с автоматическим повтором при переподключении. События буферизуются на стороне сервера и воспроизводятся при переподключении клиента, устраняя потерю событий при перебоях сети. Включает восстановление при устаревших переподключениях, повторную подписку на watch и защиту от двойных вызовов `onClientDisconnected`.

## Исправления

- **Переработка управления моделями Copilot** — Полная переработка обнаружения и синхронизации моделей GitHub Copilot. CopilotClient CLI заменён прямыми HTTP API-вызовами, добавлена 3-уровневая цепочка отката (HTTP API → CLI → статический каталог), исправлены множественные гонки при обновлении моделей. Модели без явной политики включения теперь исключаются, устаревшие модели GPT-4/3.5 отфильтровываются, а устаревшие данные больше не перезаписывают корректные списки моделей.
- **Активация контекста 1M** — Исправлено окно контекста 1M для Opus 4.6 и Sonnet 4.6 путём перехода с beta-заголовков на суффикс модели `[1m]`.
- **Windows: проверка VC++ runtime** — Усилено обнаружение vcredist с улучшенной типобезопасностью, покрытием тестами и обработкой ошибок.
- **Windows: бинарные инструменты работы с документами** — Конвертация документов (`markitdown`, `pdf-tool` и т.д.) и доступ к ресурсам навыков теперь корректно работают на Windows.

## Улучшения

- **Уведомление VC++ на Windows** — Блокирующий стартовый диалог для отсутствующих распространяемых пакетов VC++ заменён на неблокирующее внутриприложенное уведомление-тост.
- **Логирование обновления моделей Copilot** — Добавлено диагностическое логирование для помощи в устранении проблем синхронизации моделей.
