# v0.4.7 — Исправление многоуровневого разрешения навыков

Целевой релиз с исправлением критической проблемы, при которой навыки из глобальных (`~/.agents/`) и проектных (`.agents/`) директорий не вызывались, несмотря на отображение в UI.

---

## Исправления

- **Сбой навыков из глобальных/проектных директорий с ошибкой «Неизвестный навык»** — UI обнаруживал навыки из всех 3 уровней (глобальный, рабочего пространства, проектный) через `loadAllSkills`, но бэкенд SDK регистрировал только директорию рабочего пространства как плагин. Глобальные и проектные навыки отображались как доступные для вызова, но возвращали «Неизвестный навык» при использовании. Исправлено регистрацией всех 3 директорий навыков как SDK-плагинов, переработкой `qualifySkillName` для разрешения корректного префикса плагина для каждого уровня и исправлением меню упоминаний в UI для использования корректного квалификатора плагина на основе источника навыка (0fb687d5)
- **Флаги функций не включены в упакованное приложение** — `feature-flags.ts` отсутствовал в шаге копирования скрипта сборки, из-за чего упакованные сборки (особенно на Windows) не могли загрузить флаги функций времени компиляции (6268fefd, c44899cb)

## Улучшения
