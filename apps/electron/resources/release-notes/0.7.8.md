# v0.7.8 — Amazon Bedrock, контекст 1M и обработка ошибок

---

## Возможности

- **Провайдер Amazon Bedrock** — полная поддержка IAM-учётных данных для подключений Amazon Bedrock. Предварительная регистрация модуля провайдера Bedrock для предотвращения ошибок bundled dynamic-import, внедрение IAM-учётных данных как scoped subprocess env vars, принудительное использование HTTP/1.1 для совместимости с Bun/Electron.
- **Контекстное окно 1M** — Opus 4.6 и Sonnet 4.6 теперь используют контекстное окно на 1 000 000 токенов.
- **Пропуск настройки при онбординге** — пользователи теперь могут пропустить выбор провайдера во время онбординга и настроить его позже.
- **Поддержка `--base-url` в CLI** — флаг `--base-url` теперь маршрутизируется через Pi-бэкенд (тот же кодпуть, что и в десктопном UI), вместо того чтобы игнорироваться.

## Исправления

- **Ошибки сетевого прокси** — HTML-страницы ошибок Cloudflare, вызванные некорректной конфигурацией прокси, теперь показывают понятное сообщение об ошибке вместо сырого HTML. Обнаружение работает как с прокси, так и без него.
- **OAuth-обнаружение для Streamable HTTP MCP** — серверы, использующие транспорт Streamable HTTP, теперь корректно используют POST для обнаружения OAuth-метаданных, что устраняет ошибки подключения для серверов, не поддерживающих GET на `/.well-known/oauth-authorization-server`.
- **Прерывание отправки плана** — подавлены ложные сообщения об ошибках, появлявшиеся после прерывания выполнения агента через SubmitPlan.
- **Сохранение транскрипта сессии** — промежуточные сообщения (результаты инструментов, системные события) теперь сохраняются в JSONL-файл сессии, устраняя пропуски в транскрипте при перезагрузке.
- **Ограничение истории автоматизаций** — история запусков автоматизаций теперь ограничена 20 запусками на автоматизацию (1000 глобально) с runtime-компактификацией, предотвращая неограниченный рост хранилища.
- **Обобщённые сообщения об ошибках** — убрано захардкоженное «Anthropic» из сообщений об ошибках, чтобы не-Anthropic провайдеры отображали корректный текст ошибки.

## Улучшения

- **Поток учётных данных Bedrock** — единый запрос учётных данных IAM устраняет состояние гонки между внедрением env vars и JSONL-запросом аутентификации. AWS env vars извлекаются из того же результата `getPiAuth()`, который используется для init-сообщения.
- **Обработка ошибок Pi-прокси** — извлечён SDK-маппер ошибок и добавлен выделенный код `proxy_error` для типизированного соответствия ошибок между бэкендами Claude и Pi.
- **Хранилище истории автоматизаций** — история автоматизаций выделена в отдельный модуль `history-store` с компактификацией, периодическим сохранением и тестовым покрытием.

## Документация

- Добавлена справочная страница **Сетевой прокси**, описывающая конфигурацию прокси, обход через `NO_PROXY` и устранение неполадок.
