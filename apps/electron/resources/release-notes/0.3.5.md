# v0.3.5 — Claude Opus 4.6 и надёжность автообновления

Поддержка модели Claude Opus 4.6, надёжный прогресс автообновления на macOS, автоматическое изменение размера изображений и улучшения упоминания навыков.

---

## Поддержка Claude Opus 4.6

ROX теперь поддерживает **Claude Opus 4.6** (`claude-opus-4-6`) — новейшую и наиболее мощную модель от Anthropic.

### Что изменилось

- **Новая топовая модель** — Opus 4.6 вышел и теперь доступен в ROX ONE
- **Доступна везде** — страница настроек, селектор моделей чата, playground и инструмент `call_llm`
- **Opus 4.5 всё ещё доступен** — отображается как «Предыдущее поколение» для пользователей, предпочитающих его

---

## Надёжное автообновление macOS

Обновлён electron-updater до v6.8.0+, что исправляет эмиссию событий download-progress на macOS.

### Что изменилось

- **Реальный процент загрузки** — macOS теперь показывает фактический прогресс загрузки вместо неопределённого спиннера
- **~150 строк обходных путей удалено** — Резервный вариант с файловым наблюдателем macOS больше не нужен
- **Надёжное событие update-downloaded** — Обновления завершаются надёжно без зависаний

---

## Автоматическое изменение размера изображений

Большие изображения теперь автоматически изменяют размер вместо отклонения с ошибкой.

### Что изменилось

- Изображения, превышающие лимит в 8000px, **автоматически масштабируются** для соответствия ограничениям API
- Типизированные коды ошибок (`ImageValidationError`) для лучшей программной обработки
- Более понятные сообщения об ошибках, когда изменение размера невозможно

---

## Улучшения упоминаний навыков

- **Поддержка точек в ID рабочих пространств** — упоминания навыков теперь поддерживают точки (`.`) в ID рабочих пространств, в дополнение к пробелам, дефисам и подчёркиваниям
- **Поддержка пробелов в slug'ах рабочих пространств** — паттерны regex обновлены для совместимости с путями Windows
- **28 новых модульных тестов** для парсинга паттернов навыков с различными форматами ID рабочего пространства
