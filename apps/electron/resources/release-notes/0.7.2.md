# v0.7.2 — Пресеты провайдеров, система островов и исправления ошибок

---

## Возможности

- **Система островов** — извлечены `Island` и `IslandFollowUpContentView` в общий пакет `@rox-one/ui` как переиспользуемые примитивы для последующих аннотаций и встроенного расширения контента (7c4ba6af, 2432b421, 4397b284)
- **Пресет провайдера Minimax** — добавлен Minimax как встроенный пресет LLM-подключения с преднастроенным базовым URL и списком моделей (36cf45d5)
- **Пресет провайдера Kimi (Coding)** — добавлен Kimi Coding от Moonshot как встроенный пресет LLM-подключения (c59d9d8d)
- **Региональные пресеты OpenAI** — добавлены пресеты региональных API-эндпоинтов OpenAI EU (`api.openai.eu`) и US (`api.openai.us`) (bfc2be3c)
- **Уровень размышления по умолчанию на уровне приложения** — уровень размышления (off / think / max) теперь можно задать как общий настрой по умолчанию, который сохраняется между сессиями и перезапусками приложения; настройки уровня рабочего пространства имеют приоритет (f5593fb1, 7c0f2659)
- **Режим локальной dev-сборки** — новый флаг сборки `CRAFT_DEV_RUNTIME` разрешает SDK и перехватчик из монорепозитория при разработке с платформенно-зависимым fallback на bun; добавлены скрипты `electron:dist:dev:{mac,win,linux}` (506f2b02)

## Улучшения

- **Отложенные проверки SDK** — проверка Anthropic SDK и среды выполнения теперь отложена от запуска приложения до создания сессии (`strict: false` при загрузке, `strict: true` при старте сессии), что позволяет приложению открывать окна даже когда SDK/перехватчик/Bun ещё не доступны (cc70ce76)
- **Поэтапная проверка типов** — pre-commit хук теперь запускает целевую проверку типов только для пакетов, затронутых индексированными файлами, вместо полной проверки типов монорепозитория (538b12f4)
- **Тема тостов** — текст описания и кнопка закрытия тостов Sonner теперь соответствуют токенам активной темы вместо использования жёстко заданных цветов (59cfeab4, caea757f)
- **Усиление создания веток** — строгая семантика SDK-форка для новых веток; предзапрос с проверкой на непробельные символы предотвращает ошибки `invalid_request` от Anthropic на разветвлённых сессиях (9a2f5ed2)

## Исправления

- **Ошибочная классификация OAuth-ошибок Google** — `access_denied` при нажатии пользователем «Отмена» ошибочно отображался как «приложение заблокировано администратором»; исправлена логическая ошибка (`||` на `&&`), чтобы специальное сообщение появлялось только когда описание ошибки Google упоминает верификацию (f40d2e41, 0ffee7f6, c11819ef)
- **Игнорирование пользовательских иконок статуса** — имена файлов локальных иконок статуса не обнаруживались из-за отсутствующего сканирования директории; также добавлена поддержка `.webp` в шаблон имён файлов иконок (f40d2e41, 0ffee7f6, c11819ef)
- **Маршрутизация Pi-провайдера** — пользовательские эндпоинты теперь корректно разрешают поле `piAuthProvider` вместо требования выбора пресета (f40d2e41)
- **Доступность общей сессии после удаления** — удаление сессии теперь отзывает ссылку просмотра через DELETE-запрос с 5-секундным таймаутом, предотвращая публичную доступность удалённых сессий (15e4830e, c11819ef)
- **Сбой автодополнения @mention после пробела** — сопоставление файлов теперь использует поиск подпоследовательностей вместо поиска по префиксу, поэтому `app comp` корректно совпадает с `app/components/...` (c939d930)
- **Дрейф bun.lock в CI** — обновлён `bun.lock` и зафиксирована версия bun в CI для предотвращения сбоев `--frozen-lockfile` (9e9cebd2, 15e4830e)
- **Наложение баннера транспорта на TopBar** — кнопка Retry баннера переподключения скрывалась за кнопкой помощи TopBar; исправлено поднятием смещения 48px TopBar на общий родительский контейнер (71abeda4)
- **Директория сессий Pi на Windows** — файлы сессий создавались в неправильной директории на Windows из-за проблемы с разделителями путей (d3e83985)
- **OAuth subpath для Copilot** — исправлен некорректный путь OAuth-колбэка для потоков аутентификации Copilot на базе Pi (430c6f3d)
- **Диагностика ошибок подключения** — режим «только клиент» теперь логируется при запуске; сообщения об ошибках подключения включают URL сервера для упрощения отладки (ff7cf0b8)
