# v0.8.0 — Гибридный транспорт, WebUI и мобильная версия

## Возможности
- **Гибридный локальный/удалённый транспорт** — Полный гибридный транспортный уровень с унифицированным состоянием подключения рабочего пространства, обеспечивающий единообразное поведение thin-клиента и гибридного режимов. Включает проверки здоровья при открытии переключателя рабочих пространств, индикатор CloudOff для недоступных удалённых серверов и кросс-серверную передачу сессий через `invokeOnServer` со стратегией «сначала возобновить».
- **Множественные удалённые рабочие пространства** — UI селектора рабочих пространств с поддержкой нескольких удалённых рабочих пространств, с возможностью удаления через правый клик или иконку корзины при наведении. Новые удалённые серверы предлагают пользователю назвать рабочее пространство при первом подключении.
- **Доступный из браузера WebUI** — Headless-сервер теперь обслуживает полный веб-интерфейс на том же порту. Войдите с токеном сервера из любого браузера для создания сессий, управления рабочими пространствами и использования OAuth для Claude и Copilot — настольное приложение не требуется.
- **Экспорт/импорт сессий** — UI отправки в рабочее пространство для передачи сессий между серверами. Переданные сессии корректно наследуют LLM-подключение целевого рабочего пространства.
- **Мобильный WebUI** — 120% зум по умолчанию, масштабирование touch-целей 1.3x, предотвращение авто-зума iOS Safari и адаптивный макет с использованием container queries. Улучшено поведение кнопки «назад», заголовки панелей, компактный режим и адаптивность карточек.

## Улучшения
- **Усиление безопасности цепочки поставок** — Список `trustedDependencies` блокирует несанкционированные установочные скрипты. `--frozen-lockfile` применяется во всех CI-workflow для предотвращения незаметного дрейфа зависимостей.
- **Обзор директорий удалённого сервера** — Канал `server:homeDir` позволяет обозревателю директорий работать с путями удалённого сервера.
- **UX подключения к серверу** — Переписан поток с двумя различными путями: подключение к существующему рабочему пространству или создание нового.
- **Поддержка самоподписанных TLS-сертификатов** — Удалённые `wss://`-подключения принимают самоподписанные сертификаты для локальных и intranet-разворачиваний.

## Исправления
- **Разрешение упоминаний навыков/источников** — Упоминания теперь разрешаются в семантические маркеры вместо удаления, сохраняя контекст в промптах агента.
- **Запуск подпроцесса сервера** — Подпроцесс-серверы собираются до запуска headless-сервера и используют `process.execPath` вместо bare `bun` для определения пути к среде выполнения.
- **Краш сервера при прерывании** — Перехват необработанных отклонений промисов, которые ранее приводили к крашу сервера при прерывании сессии.
- **Переключение рабочих пространств WebUI** — `switchWorkspace` был no-op, что ломало доставку push-событий; теперь корректно маршрутизирует события.
- **Отклонение TLS в WebUI** — WebUI обслуживается на порту WebSocket, предотвращая отклонение TLS-сертификата браузерами.
