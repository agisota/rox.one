# v0.7.3 — Стабильность OAuth, фоновые задачи и генерация заголовков

---

## Возможности

- **UI фоновых задач** — карточки инструментов теперь показывают спиннер во время выполнения фоновых задач (Agent, Task), обеспечивая чёткую визуальную обратную связь по активной работе (20352706)
- **Улучшенная генерация заголовков** — заголовки сессий теперь генерируются с использованием расширенного выбора сообщений и учёта языка, что даёт более точные и естественные заголовки на разных языках (835ea942, 55b7830c, 46975ac7)
- **Бейджи исключающих фильтров** — Alt/Option-клик по элементам фильтра теперь добавляет их как исключающие фильтры с предпросмотром бейджа при зажатой клавише Alt (fce25e23, 8a2ebbee)
- **Тесты жизненного цикла автоматизаций** — `--validate-server` теперь включает тесты жизненного цикла автоматизаций, выявляя регрессии в конвейере автоматизаций (19fe11e9)

## Улучшения

- **Конвертация MCP-схем** — `oneOf`, `anyOf`, `allOf` и вложенные объекты теперь корректно обрабатываются при конвертации схем MCP-инструментов, что уменьшает количество ошибок «неожиданный параметр» от провайдеров. Частично устраняет проблему (64ae9d69)
- **Разделение пресетов Minimax** — провайдер Minimax теперь разделён на отдельные варианты Global и CN (Китай) с корректными региональными эндпоинтами (d626f732, 5ba42ca3)
- **Разрешение упоминаний @file** — упоминания файлов во вводе чата теперь обёрнуты в семантические маркеры, чтобы агент мог корректно читать и разрешать их (15d20c1d)
- **Автосоздание меток** — метки, на которые ссылаются автоматизации, теперь автоматически создаются при отсутствии, предотвращая тихие сбои (76306c0b)
- **Мульти-ОС валидация в CI** — рабочий процесс `validate-server` теперь запускается на матрице macOS + Windows + Linux с включённым fail-fast (32f4c91e)
- **Учёт языка при генерации заголовков** — заголовки лучше отражают язык беседы. Частично устраняет проблему (835ea942, 1892b4c9)

## Исправления

- **Ложные повторные OAuth-авторизации** — исправлено состояние гонки, при котором источники вызывали ненужные потоки повторной авторизации после успешного обновления токена, что приводило к прерыванию соединений и дублированию запросов авторизации (b98a2b2d, e101301b, a3264b02, 76bb5388, d8632e30)
- **Отключение MCP-источников при обновлении токена** — MCP-источники теперь корректно переподключаются при обновлении OAuth-токена вместо того, чтобы оставаться в отключённом состоянии (b1b515b2)
- **Блокировка меню повторной авторизации** — исправлено состояние гонки, при котором меню повторной авторизации могло зависнуть в заблокированном состоянии, блокируя дальнейшее взаимодействие с пользователем (34dfc91f)
- **Состояние гонки в MCP-транспорте** — исправлена гонка при последовательных SDK-запросах, которая могла повредить транспортный уровень MCP (f7d3f902)
- **Зависание активностей Agent/Task** — активности фоновых инструментов (Agent, Task) больше не застревают в состоянии «выполняется» после завершения или ошибки базовой задачи (6d49ae82)
- **Утечка памяти фоновых задач** — извлечены вспомогательные функции инструментов и исправлена утечка памяти в обработке уведомлений фоновых задач (6979d063)
- **Регрессия удаления преамбулы** — итеративное удаление преамбулы теперь обрабатывает краевые случаи при лингвистической очистке и надёжнее фильтрует низкосигнальный контент (1892b4c9)
- **Дублирование импорта** — удалён дублирующийся импорт `Message` в `turn-utils`, оставшийся после слияния (0aa7045f)
