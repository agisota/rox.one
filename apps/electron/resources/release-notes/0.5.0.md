# v0.5.0 — Используйте любого LLM-провайдера

---

## Новые подключения провайдеров

- **Codex и ChatGPT Plus** — войдите с подпиской ChatGPT Plus или Pro для использования моделей OpenAI Codex напрямую
- **GitHub Copilot** — подключите подписку GitHub Copilot с помощью аутентификации по коду устройства в один клик
- **Google AI Studio** — используйте модели Gemini с вашим API-ключом Google AI Studio. Поддержка первого класса с встроенным нативным поиском Google.
- **OpenRouter, Ollama и пользовательские эндпоинты** — используйте свой API-ключ для любого совместимого с OpenAI провайдера — OpenRouter, Ollama или любой пользовательский эндпоинт
- **Поисковый выбор моделей** — когда у провайдера много моделей, выбирайте из трёх уровней (Лучшие, Сбалансированные, Быстрые) через поисковые выпадающие списки с умными настройками по умолчанию

## Улучшения

- **Иконки провайдеров** — настройки подключения теперь показывают логотип и имя вышестоящего провайдера, чтобы вы могли отличать подключения с первого взгляда
- **MCP-источники работают со всеми провайдерами** — подключённые MCP-источники теперь проксируются к не-Anthropic бэкендам, поэтому такие инструменты, как Linear, GitHub и Slack, работают независимо от того, какую LLM вы используете
- **Редизайн списка сессий** — переиспользуемые примитивы entity-list для более чистого и согласованного UI
- **Метаданные навыков** — `requiredSources` и `icon` теперь отображаются на странице информации о навыке и принимаются при валидации
- **Фильтрация моделей Copilot** — отображаются только модели, разрешённые политикой вашей организации (92200095)

## Исправления

- **Цвета иконок статуса** — восстановлены цвета в контекстном меню правого клика (6fa9b85b)
- **Пути Windows** — исправлена обработка обратных слешей при извлечении slug рабочего пространства (3d22e75f)
- **Сохранение уровня модели** — выбранные модели больше не молча перезаписываются настройками провайдера по умолчанию (1ddfe96f)
- **Сборка Windows** — исправлено разрешение нативных модулей (koffi), OOM в Vite и упаковка файлов перехватчика для надёжных сборок Windows (0ae044a2, ff791b03, 258e3e39)
- **Среда выполнения Windows** — обновлён встроенный Bun 1.3.5 до 1.3.9 для исправления сбоя на Windows (c4f46abc)
- **Pi SDK 0.55.0** — обновлён Pi SDK для исправления зависаний потоковой передачи и некорректного JSON частичных чанков OpenAI (718d4ad6)
- **Скрипты установки** — исправлены скрипты установки для Windows и macOS/Linux для работы без эндпоинта версий; скрипты теперь напрямую получают YAML-манифесты

## Безопасность

- **Усиление web-fetch** — защита от SSRF, неограниченных чтений и других краевых случаев (3904a786)
- **Режим Explore** — заблокированы `find -exec` и аналогичные векторы обхода оболочки

## Инфраструктура

- **Миграция на Cloudflare** — мигрированы R2-бакет и Workers на новый аккаунт Cloudflare (55bb899e)
- **CI** — сборки Windows теперь используют раннер `windows-latest` (c4ac80a6)

## Критические изменения

- **Консолидация бэкендов Codex/Copilot** — отдельные бэкенды Codex и Copilot заменены унифицированным бэкендом Pi SDK. Существующие подключения Codex и Copilot потребуется перенастроить через новый поток онбординга.
- **Удалён Claude OAuth через Pi** — опция использования подписки Claude через бэкенд Pi удалена для соответствия условиям использования Anthropic. Используйте прямое подключение Claude Pro/Max.
