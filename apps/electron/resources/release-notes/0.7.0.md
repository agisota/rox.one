# v0.7.0 — ROX ONE CLI, сервер без GUI и масштабная переработка архитектуры

---

## ROX ONE CLI

Новый терминальный клиент для запуска сессий агента из командной строки — без GUI.

- **`rox-cli run`** — запуск сессий агента прямо из терминала с вводом промптов и потоковым выводом
- **Поддержка нескольких провайдеров** — выбор между несколькими LLM-подключениями и моделями из CLI
- **Поддержка рабочих пространств CI** — программное создание и управление рабочими пространствами для автоматизированных процессов
- **`--validate-server`** — проверка конфигурации сервера с форматированным диагностическим выводом
- **Обновлённый спиннер** — остаётся видимым во время размышлений агента для лучшего UX в терминале

### Почему это важно
Это первый шаг к полному программному управлению ROX ONE. Скрипты для рабочих процессов агента, интеграция в CI/CD конвейеры или запуск сессий без GUI на удалённых машинах.

## Сервер без GUI

Запуск ROX ONE как автономного сервера без десктопного приложения Electron — развёртывание на любой машине, подключение с любого клиента.

- **Автономный сервер на Bun** — точка входа без GUI, работающая без Electron, с исключённым sharp из бандла
- **Поддержка TLS** — безопасные удалённые подключения с автоматической обработкой сертификатов
- **Мультиплатформенный скрипт сборки** — упаковка дистрибутива для Linux, macOS и Windows
- **Удалённый режим через переменные окружения** — настройка работы без GUI через переменные окружения
- **Состояние подключения тонкого клиента** — статус WebSocket-подключения и баннеры ошибок в клиенте

### Почему это важно
Развёртывание ROX ONE на удалённом сервере с возможностью подключения из любого места. Запуск на домашнем сервере, облачной ВМ или CI-раннере. Режим тонкого клиента означает, что браузерный UI остаётся лёгким, а вся тяжёлая обработка происходит на стороне сервера.

## Транспорт и архитектура

Весь уровень межпроцессного взаимодействия перестроен с нуля.

- **Только WebSocket RPC** — Electron IPC заменён на WebSocket RPC для всей коммуникации, что обеспечивает удалённую работу
- **Двунаправленный RPC** — типизированные push-события, управление OAuth-токенами на стороне сервера
- **Извлечение server-core** — 14 базовых обработчиков, управление сессиями, сервисы и доменные утилиты извлечены в `@rox-one/server-core` для переиспользования в Electron и headless
- **Формализация протокола** — каналы, DTO и карты событий извлечены в `@rox-one/shared/protocol` с тестами стабильности формата передачи
- **Очистка split-brain** — удалены мёртвые копии обработчиков/сессий, добавлены `IWindowManager` и реклама каналов

### Почему это важно
Это фундамент, который делает возможными CLI, headless-сервер и будущие удалённые/мобильные клиенты. Архитектура теперь чётко разделена между переиспользуемой логикой server-core и платформенно-зависимыми оболочками (Electron, Bun-сервер, CLI).

## Редактор и UI

- **Взаимодействия с блоками** — поддержка изображений tiptap, слеш-меню для вставки блоков контента и fallback темы
- **Цепочка fallback моделей** — автоматический переход между моделями при недоступности одной из них с улучшенным оформлением плейграунда
- **Браузерные инструменты** — улучшенное управление жизненным циклом, компоненты бейджей метаданных и границы безопасного режима

## Безопасность

- **Блокировка незашифрованных WebSocket** — удалённые подключения теперь требуют `wss://` (TLS); обычный текст `ws://` отклоняется
- **Генерация сертификатов** — исправлено имя кривой в скрипте генерации сертификатов

## Исправления

- **Сохранение 3-уровневых моделей Pi** — исправлено сохранение/восстановление моделей для уровней Best/Balanced/Fast
- **Фокус ввода в мульти-панелях** — исправлена область фокуса и обработка гонок, из-за которых нажатия клавиш перескакивали между панелями
- **Базовый URL Pi Azure** — базовый URL Azure передаётся в подпроцесс Pi
- **Маршрутизация deep-link** — исправлено целевое назначение deep-link на клиенте с регрессионными тестами
- **Расширение split-turn** — исправлены коллизии ключей расширения TurnCard
- **Ложное срабатывание Explore mode** — синтаксис стрелок node (`=>`) больше не помечается как операция записи
- **Цикл онбординга** — исправлено постоянное отображение онбординга и тихие сбои сохранения подключений
- **Листинг моделей Copilot** — используется GitHub OAuth-токен для листинга моделей CopilotClient
- **Наложение бейджей меток** — предотвращено преждевременное наложение бейджей меток в UI
- **Усиление транспорта** — усилены маршрутизация запуска WebSocket, кодек и изоляция наблюдателя сессий
- **Исправления OAuth** — открытие браузера OAuth локально через `shell.openExternal`, принятие пути `/oauth/callback`
- **Полезные нагрузки событий** — согласованная форма `sources:changed` и `skills:changed`
- **Защита глобальной конфигурации** — обеспечение существования `config.json` перед регистрацией обработчика
