# v0.8.5 — Многоязычная поддержка и обновление Pi SDK

## Возможности
- **Многоязычная поддержка (i18n)** — Весь UI теперь локализован. Поставляется с **английским**, **испанским (es)**, **упрощённым китайским (zh-Hans)** и **японским (ja)** — более 1 050 переведённых строк, покрывающих каждую страницу, меню, уведомление, подсказку и диалог. Переключайте язык в Настройки → Внешний вид. Заголовки сессий и ответы AI также следуют выбранному языку.
- **Обновление Pi SDK (0.56.2 → 0.66.1)** — Крупное обновление, охватывающее несколько upstream-релизов. Модели вроде **GLM 5**, **GLM 5.1** и **Minimax 2.7** теперь должны работать благодаря upstream-исправлениям. Также включает: ограничение частоты Bedrock больше не ошибочно определяется как переполнение контекста, обнаружение HTTP 413 от Anthropic для компакции/повторных попыток, поддержка потоковой передачи инструментов Z.ai, исправления потоковой передачи OpenAI и исправление обрезки вывода bash.

## Улучшения
- **Канонический реестр локалей** — Добавление нового языка теперь требует изменения одного файла. Реестр автоматически выводит все коды языков, отображаемые имена, ресурсы i18n и локали date-fns.
- **Единая настройка языка** — Удалено дублирующееся текстовое поле «Язык» из Настроек. Выпадающий список языка во Внешнем виде теперь единственный источник истины как для языка UI, так и для языка ответов AI.
- **Инструментарий разработчика i18n** — Pre-commit hook перехватывает захардкоженные английские строки в staged `.tsx`-файлах. Тест чётности локалей гарантирует, что все переводы остаются синхронизированными. Навык `localize-agents` автоматизирует добавление новых языков.

## Исправления
- **Потеря сообщений сессии после пробуждения ПК** — После спящего режима/пробуждения, если подпроцесс сервера завершался, сообщения сессии терялись, а область содержимого отображалась пустой. Исправлено логикой повторных попыток, fallback-ом ленивой загрузки и повторным запросом сообщений активной сессии.

## Известные ограничения
- **Ответы headless-сервера на английском** — При использовании WebUI, подключённого к удалённому headless-серверу, UI полностью локализован (на основе языка браузера), но AI-агент всё ещё отвечает на английском. Заголовки сессий также генерируются на английском. Поддержка языка для каждого клиента на headless-сервере будет добавлена в будущем релизе.

## Критические изменения
- Текстовое поле «Язык» в Настройках удалено. Используйте выпадающий список языка в Настройки → Внешний вид. Существующие файлы `preferences.json` с полем `language` безопасно игнорируются.
