# v0.5.1 — Автоматизации

---

## Автоматизации

Автоматизации (ранее hooks/tasks) теперь имеют полноценный UI управления — больше не нужно редактировать JSON-файлы вручную.

- **Боковая панель** — просмотр, поиск и фильтрация автоматизаций по типу (Все, По расписанию, Событийные)
- **Включение / Отключение** — переключение отдельных автоматизаций без их удаления
- **Дублирование и удаление** — клонирование существующей автоматизации или её удаление с поддержкой массовых операций при множественном выборе
- **История выполнения** — каждая автоматизация записывает успехи/сбои на таймлайн, просматриваемый на странице деталей
- **Редактирование в приложении** — кнопка редактирования на странице деталей автоматизации для быстрых изменений
- **Тестовый запуск** — ручной запуск автоматизации для проверки корректности работы до наступления реального события

### LLM-подключение и модель для действий типа Prompt

Действия типа prompt теперь могут указывать, какой ИИ-провайдер и модель использовать для создаваемой сессии:

```json
{
  "type": "prompt",
  "prompt": "Review overnight @github pull requests",
  "llmConnection": "my-openrouter",
  "model": "anthropic/claude-sonnet-4.6"
}
```

Значение `llmConnection` — это slug LLM-подключения из настроек AI. Валидация выявляет отсутствующие подключения (ошибка) и несоответствия модели/провайдера (предупреждение) до выполнения.

### Формат конфигурации

Канонический файл конфигурации теперь `automations.json` с форматом версии 2. Устаревший `hooks.json` больше не загружается — переименуйте и обновите для продолжения использования существующих автоматизаций.

## Улучшения

- **API-источники для всех провайдеров** — API-источники (Slack, Gmail) теперь доступны в пуле MCP, что делает их доступными для не-Anthropic бэкендов, таких как Copilot, Codex и Gemini
- **Метаданные навыков** — поля `requiredSources` и `icon` теперь отображаются на странице информации о навыке
- **Сортировка списка автоматизаций** — сортировка по последнему выполнению для быстрого доступа к активным автоматизациям
- **Событие агента `Setup`** — новый тип события для автоматизаций инициализации агента
- **Отображение года в cron** — отображается год в предпросмотре следующего запуска, когда выполнения охватывают несколько лет

## Исправления

- **Упоминания файлов** — упоминания файлов и папок в сообщениях чата больше не удаляются; теперь они разрешаются в абсолютные пути для агента
- **Директория Config watcher** — ConfigWatcherManager больше не создаёт директории `sources/` и `skills/` внутри рабочей директории сессии вместо корня рабочего пространства
- **Обработчик тестирования автоматизаций** — ручной «Тест» теперь разрешает `@mentions`, включает источники и передаёт `llmConnection`/`model`/`permissionMode` (тот же путь кода, что и у планировщика)
- **Скрипты установки** — прямое получение YAML-манифестов вместо зависимости от эндпоинта версий; несколько исправлений совместимости с PowerShell 5.1

## Критические изменения

- **Удаление устаревшей конфигурации** — `hooks.json` больше не читается, автоматическая миграция отсутствует. Если у вас были существующие хуки, их необходимо пересоздать как автоматизации в новом формате `automations.json`. Самый простой способ — описать свои рабочие процессы агенту — например, «настрой ежедневный утренний брифинг в 9 утра» — и он создаст конфигурацию за вас.
