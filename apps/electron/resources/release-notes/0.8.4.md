# v0.8.4 — Универсальный OAuth для источников, отправка в рабочее пространство и DevTools

## Возможности
- **Универсальный OAuth для API-источников** — API-источники теперь могут использовать стандартную OAuth-авторизацию. Включает автоматическое обнаружение эндпоинтов по RFC 9728 (метаданные сервера авторизации OAuth), поэтому нужно только указать URL эмитента, а остальное разрешится автоматически.
- **Отправка в рабочее пространство** — Новое действие для обмена источниками, навыками и автоматизациями между рабочими пространствами одним кликом. Поддерживает как индивидуальные, так и пакетные операции для перемещения нескольких ресурсов сразу.
- **DevTools в упакованном приложении** — Инструменты разработчика теперь доступны в production-сборках через Вид → Переключить инструменты разработчика, что упрощает отладку без dev-сборки.

## Улучшения
- **Документация OAuth** — Добавлены руководства по настройке универсального OAuth и автообнаружению в онлайн-документации и руководстве по источнику GitHub.

## Исправления
- **Инструменты сессий недоступны через Pi** — Инструменты самоуправления сессиями (`set_session_labels`, `set_session_status` и др.) теперь корректно работают при маршрутизации через путь агента Pi.
- **Экран остаётся включённым навсегда** — Исправлено: экран оставался включённым даже после отключения настройки «Не выключать экран».
- **Трансляция payload для удалённой маршрутизации** — `workspaceId` внутри вложенных объектов payload теперь корректно транслируется для маршрутизации удалённых рабочих пространств.
- **Загрузка автоматизаций в WebUI** — Автоматизации теперь загружаются через серверный RPC вместо прямого чтения файлов, что исправляет сбои на удалённых/Docker-установках.
- **Неработающие кнопки в карточках ошибок** — Кнопки «Настройки» и «Повторить» в карточках ошибок теперь корректно подключены и функциональны.
- **Подсказки уровней моделей привязаны к Anthropic** — Поповер селектора моделей теперь использует учётивающие провайдера подсказки уровней вместо специфичных для Anthropic алиасов, исправляя проблемы отображения для не-Anthropic провайдеров.
- **Получение OAuth-токена для API-источников** — Исправлено: `getToken` не был подключён для API-источников с универсальным OAuth в SessionManager.
- **Сообщения об ошибках OAuth** — Улучшено сообщение об ошибке при отсутствии блока конфигурации `api.oauth` у источника.
- **Импорт ресурсов на Linux** — ConfigWatcher теперь уведомляется после импорта ресурсов, исправляя совместимость с `fs.watch` на Linux, где импортированные ресурсы не отображались до перезапуска.
- **Заметки о релизе на Docker/remote** — Заметки о релизе теперь используют fallback `~/.rox/release-notes/`, когда путь ресурсов Electron недоступен.
- **Страница «О программе» в WebUI** — Диалог «О программе» теперь показывает версию сервера, скрывает нерелевантную кнопку «Проверить сейчас», а «Что нового» корректно работает в WebUI.
