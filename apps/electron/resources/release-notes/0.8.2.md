# v0.8.2 — OAuth в WebUI, управление браузером и надёжность поиска

## Возможности
- **Переключатель встроенного браузера** — Добавлена настройка для отключения встроенного `browser_tool`, если вы предпочитаете внешний MCP-браузер или реальную авторизованную браузерную сессию.
- **Унифицированный OAuth источников в WebUI** — Браузерная аутентификация источников теперь маршрутизируется через стабильный relay-callback с серверным завершением, что делает OAuth-потоки WebUI более надёжными для Google, Microsoft, Slack и источников на базе MCP.
- **Ресурсы для установки Web UI как приложения** — Добавлены favicon, manifest и Apple touch icon для браузерного UI.
- **Ускорение файловых операций** — Добавлено покессионное кэширование для дорогостоящих файловых операций, снижающее повторные накладные расходы во время ходов агента.

## Улучшения
- **Усиление аутентификации WebUI** — Замена самодельного JWT-пути на `jose`, хэширование паролей через `argon2id` и добавлен глобальный ограничитель частоты для auth-эндпоинтов headless/WebUI.
- **Документация headless-развёртывания** — Расширена документация собственного сервера с руководством по Cloudflare Tunnel и уточнением устранения неполадок Docker/Home-директории.
- **Визуальная полировка** — Улучшена контрастность тёмного режима в приложении и выравнены стили со значениями темы по умолчанию.
- **Очистка метаданных заметок о релизе** — Исправлены отсутствующие заголовки версий во внутриприложенных заметках о релизе «Что нового».

## Исправления
- **Несовпадение редиректа Google OAuth в WebUI** — Исправлено `redirect_uri_mismatch` в WebUI путём использования стабильного callback для провайдера и relay реальной цели через state.
- **Надёжность поиска и согласованность счётчиков** — Исправлено несколько регрессий поиска: старые сессии сопоставляются надёжнее, кросс-узловые совпадения внутри блоков кода с подсветкой синтаксиса находятся корректно, счётчики бейджей теперь соответствуют реальным подсвеченным совпадениям, а поиск в WebUI/remote ухудшается более плавно при недоступности `ripgrep`.
- **Стабильность подсветки поиска** — Переработан рендеринг подсветки вокруг CSS Custom Highlight API для устранения устаревших ссылок, отменённых RAF и предупреждений таблиц стилей.
- **Проводка OAuth-callback** — Подключены `oauthCallbackDeps` через server bootstrap, так что завершение callback WebUI стабильно достигает нужного серверного потока.
- **Фильтрация вложений** — Расширен фильтр base64 для перехвата миниатюр вложений и избежания ложноположительных совпадений поиска.
- **Исправления маршрутизации и ссылок** — Исправлен маркетинговый маршрут и обновлены ссылки загрузки для соответствия текущему именованию артефактов.
