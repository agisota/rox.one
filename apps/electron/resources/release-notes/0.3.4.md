# v0.3.4 — Конвенция Skills и улучшения сборки Windows

Поддержка конвенции `.agents/skills` и значительное улучшение надёжности сборки Windows.

---

## Поддержка конвенции `.agents/skills`

Навыки теперь можно размещать в директории `.agents/skills` в соответствии с новым кросс-инструментным стандартом.

### Что нового

**Кросс-инструментовая совместимость**
Поместите свои навыки в `~/.agents/skills/`, и они будут работать с ROX ONE, Codex, Gemini и другими инструментами, поддерживающими эту конвенцию. Существующие навыки рабочего пространства продолжают работать как раньше.

---

## Исправление приоритета конфигурации темы

Настройки темы в файле конфигурации теперь корректно переопределяют localStorage.

### Как это работает

**Конфигурация как источник истины**
Когда вы устанавливаете `colorTheme` в `~/.rox/config.json`, он теперь корректно имеет приоритет над кэшированной темой в localStorage. Это позволяет агентам изменять тему через документированные пути конфигурации.

## Улучшения

### Обработка ошибок
- Понятные сообщения для ошибок AI-провайдеров (internal server error, api_error) с указанием проверить страницу статуса
- Поддержка пользовательских моделей теперь корректно работает в саммаризации — использует `resolveModelId()` для всех внутренних вызовов моделей

### Навыки
- Исправлен формат упоминания навыков — `@`-упоминания теперь генерируют корректный формат workspace-алиасов вместо сырых UUID
- Улучшения поведения прокрутки Steps — «Steps Completed» теперь прокручивает к самому последнему шагу

### UI
- Предотвращение рендеринга EditPopover за пределами viewport
- Исправлено отображение времени «60m» вместо «1h»
- Сохранение переносов строк в многострочных пользовательских сообщениях (Shift+Enter)

### Linux (Hyprland/Wayland)
- Таймаут-резерв при закрытии окна для Hyprland/Wayland, где IPC может не работать

## Исправления

- **Навыки**: исправлена генерация неверного формата ID в `@`-упоминаниях навыков (UUID вместо workspace-алиаса)
- **UI**: EditPopover теперь остаётся в пределах viewport
- **Linux**: таймаут-резерв при закрытии окна для окружений Hyprland/Wayland

## Сборка и CI (Windows)

Значительные улучшения надёжности сборки Windows:

- Пакеты объединяются в bundle вместо пометки как external для лучшей совместимости
- Использование `--linker=hoisted` при `bun install` на Windows для избежания `.bun`-симлинков
- SDK объединяется через extraResources для обхода исключения node_modules
- Добавлен `bun.lock` для единообразных кроссплатформенных сборок
- Исправлено имя app-bundle в afterPack.cjs
- Множественные исправления обработки симлинков при упаковке Windows
