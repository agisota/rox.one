# v0.4.8 — Инструмент LLM и исправление плагинов

---

## Возможности

- **Инструмент `call_llm`** — вызов вторичной LLM для целевых подзадач: суммаризация, классификация и структурированное извлечение. Поддерживается на всех трёх бэкендах: Claude, Codex и Copilot. Работает как с API-ключом (полный функционал), так и с OAuth (базовый функционал). Поддерживает параллельные вызовы, вложения файлов и структурированный JSON-вывод (eb0a3476, fa91af4c, b6e5c85f)

## Исправления

- **Разрешение имён плагинов** — навыки не разрешались, когда имя директории рабочего пространства не совпадало с именем плагина SDK. Теперь считывается фактическое имя плагина из `.claude-plugin/plugin.json` (b7904cb7)
- **Горячая перезагрузка навыков** — добавление навыка рабочего пространства приводило к исчезновению глобальных и проектных навыков до перезапуска. Все пути перезагрузки теперь используют `loadAllSkills` для возврата полного трёхуровневого списка (4172fd82)
- **Состояние гонки в очереди событий Codex** — результаты инструментов и текст ассистента могли теряться, когда асинхронные обработчики `item/completed` всё ещё выполнялись при наступлении `turn/completed`. Исправлено откладыванием завершения очереди до окончания всех обработчиков (fa91af4c)

## Internal

- Copilot `runMiniCompletion` теперь функционален — включает генерацию заголовков на бэкенде Copilot (fa91af4c)
- Адаптер событий Copilot подавляет события reasoning/intent (fa91af4c)
- Бейдж модели `call_llm` отображается в строках активности TurnCard (8bb4bcd2)
