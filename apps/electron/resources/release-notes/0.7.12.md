# v0.7.12 — Amazon Bedrock, расширенный кэш промптов и сервер Docker

## Возможности
- **Поддержка Amazon Bedrock** — Полная аутентификация Bedrock (IAM-учётные данные, bearer-токен, переменные окружения) с автоматической нормализацией ID моделей к US cross-region inference-профилям. Подключения Bedrock маршрутизируются через Pi SDK для надёжного разрешения моделей.
- **Расширенный кэш промптов** — Новая настройка для кэширования промптов на 1 час вместо 5 минут. Снижает стоимость для длительных сессий на моделях Claude через Anthropic API. (Настройки → AI → Производительность)
- **Docker-сервер headless** — Мультиплатформенная сборка Docker для запуска ROX как headless-сервера с автоматическим разрешением зависимостей.

## Улучшения
- **Перемещён переключатель контекста 1M** — Настройка «Расширенный контекст (1M)» перенесена из Рабочее пространство → Дополнительно в AI → Производительность для лучшей обнаруживаемости.
- **Документация headless-сервера** — Переписано руководство по настройке с шагами установки, конфигурациями запуска и требованиями TLS.

## Исправления
- **ID моделей Bedrock** — Исправлены ошибки `400 ValidationException` и ошибки inference-профилей путём маппинга bare Anthropic ID моделей на нативные Bedrock US inference-профили (`us.anthropic.claude-*`). Также исправлена миграция хранилища для некорректно настроенных подключений с `providerType: 'bedrock'`.
- **Ошибка сборки Bedrock** — Исправлено `Cannot find module './amazon-bedrock.js'` путём регистрации модуля провайдера Bedrock для обеих копий Pi SDK в bundled-выводе.
- **Контекст ветвления** — При сбое ветвления на уровне SDK родительский разговор теперь суммируется через мини-модель как резервный контекст вместо тихой потери истории.
- **Схемы инструментов MCP** — Удалено свойство `$schema` из внешних схем ввода MCP-инструментов, вызывавших ошибки валидации.
- **Состояние гонки компакции Pi SDK** — Защита от состояния гонки автокомпакции в подпроцессе Pi SDK.
- **URL MCP-серверов** — Прекращено некорректное добавление `/mcp` к URL MCP-серверов.
- **Очистка перехватчиков** — Сетевой перехватчик теперь активно удаляет заголовки cache TTL и context-1M beta для предотвращения конфликтов со специфичной обработкой провайдеров.
