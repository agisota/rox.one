# v0.8.10 — Шлюз сообщений, восстановление Opus 4.6 и исправление расширенного контекста

## Возможности
- **Шлюз сообщений — Telegram и WhatsApp** — Новый шлюз на уровне воркспейса позволяет привязывать внешние чаты (личные сообщения Telegram, контакты WhatsApp) к сессиям ROX. Входящие сообщения управляют агентом, а вывод агента отображается обратно в чате. Три режима ответа: `progress` (по умолчанию — одна развивающаяся цепочка), `streaming` (редактирование в реальном времени), `final_only` (молчание до завершения). Telegram поддерживает вложения фото/документы/голос/видео/аудио (лимит 20 МБ). WhatsApp работает в подпроцессе-воркере на базе Baileys, изолируя его глобальное состояние от основного процесса. Включает новую страницу **Settings → Messaging V2** с плитками для каждой платформы, диалогами привязки, UI подключения через QR/код и статусными атомами.
- **Opus 4.6 восстановлен как выбираемая модель** — Claude Opus 4.6 вернулся в выбор моделей alongside 4.7. Пользователи на Tier 1–3, получающие "Invalid Request" на крупных проектах с 4.7, могут вернуться к 4.6 без ручного редактирования конфигов.
- **Новый уровень размышления `xhigh`** — Добавляет промежуточный уровень размышления между `high` и `max`.
- **Вывод stderr pi-agent отображается при ошибках тестирования LLM-подключения** — Неудачные тесты LLM-подключения теперь показывают underlying stderr pi-agent, чтобы ошибки конфигурации (отсутствующий бинарник, проблемы аутентификации) можно было отладить без поиска по логам.

## Улучшения
- **Перечисление `ThinkingLevel` создано из единого кортежа** — `ThinkingLevel` и его Zod-схема теперь порождаются из одного кортежа-источника-истины, предотвращая расхождение между типом и runtime-валидатором.
- **Полировка UI сообщений** — Приглушён стиль подсказок в диалогах и основные кнопки подключения; пункт меню WhatsApp "Forget Device" переименован в "Disconnect" для ясности.
- **Полировка self-chat в WhatsApp** — Ответы в self-chat помечаются префиксом 🤖, чтобы сообщения бота визуально отличались от ваших; теперь поддерживаются контакты с LID (длинными идентификаторами); встроена информация о происхождении сборки для верификации bundled-воркера во время выполнения.
- **Надёжность Docker-сборки** — Heap Node увеличен до 4 ГБ для сборки webui Vite, чтобы избежать OOM на минимальных CI-образах.
- **Верификация WhatsApp-воркера в CI** — Бандл WhatsApp-воркера теперь собирается в CI и проверяется в релизных артефактах, чтобы повреждённые бандлы не попадали в релизы.

## Исправления
- **Расширенный контекст (1M) теперь подключается вручную** — `enable1MContext` ранее по умолчанию был `true`, что вызывало ошибки 400 "Invalid Request" на крупных проектах для пользователей direct-API на Tier 1–3 (бета `context-1m-2025-08-07` требует Tier 4+). Новые установки по умолчанию отключены; пользователи подключают через *AI Settings → Performance → Extended Context (1M)*. Существующие сохранённые значения сохраняются. Путь ошибки `invalid_request` теперь специализируется на подсказках 1M-контекста и показывает сообщение "1M Context Not Available" с действием Settings вместо вводящего в заблуждение отката "remove attachments".
- **`spawn_session` теперь раскрывает `~` в `workingDirectory`** — Ранее тильда передавалась буквально в дочерний SDK, что вызывало ошибку "cli.js not found".
- **`set_session_labels` теперь принимает метки со значениями (`id::value`)** — Резолвер сравнивал каждый ввод как атомарный ID, поэтому `"priority::3"` или `"parent-task::TASK-123"` всегда попадали в "unknown label", даже если базовая метка была сконфигурирована с `valueType`. Вводы со значениями теперь парсятся перед сопоставлением; значения также проверяются на соответствие объявленному типу (`number`, `date`, `string`), а сообщение об отказе объясняет точную причину для каждой записи.
- **Модели Google по умолчанию переупорядочены** — Тест подключения теперь выбирает стабильную модель по умолчанию вместо той, которая может быть с ограничением скорости или устаревшей.
- **Восстановление после отказа codex-mini в pi-agent** — `pi-agent-server` теперь корректно восстанавливается, когда codex-mini отклоняет запрос на ChatGPT-auth Codex, вместо оставления сессии в сломанном состоянии.
- **Аннотации: закрытие follow-up island отложено после grace-периода enter-анимации** — Исправлена гонка, при которой follow-up island закрывался до завершения его enter-анимации, вызывая визуальные артефакты.
- **Формат передачи данных при восстановлении Opus 4.6** — Opus 4.6 теперь передаётся как объект `ModelDefinition` (а не как строка), чтобы потребители connection-list корректно проходили проверку типов.
