# v0.7.5 — Сетевой прокси, вебхуки и исправления сообщества

---

## Возможности

- **Поддержка сетевого прокси** — настройка HTTP/HTTPS-прокси с правилами обхода прямо из настроек приложения. Движок прокси маршрутизирует трафик через экземпляры `undici` ProxyAgent, учитывает правила `NO_PROXY` и настраивает как Node-, так и Electron-браузерные сессии.
- **Действия вебхуков для автоматизаций** — автоматизации теперь могут отправлять HTTP-вебхуки с настраиваемой аутентификацией, form-payloads, захватом ответов, повторным воспроизведением и персистентными повторами с экспоненциальной задержкой.
- **Gemini 3.1 Flash Lite** — добавлен в предпочтительные значения по умолчанию Google AI Studio.
- **Удаление элементов из истории рабочих директорий** — кнопка X, видимая при наведении на каждую запись недавней директории, для удаления из истории.

## Улучшения

- **Усечение истории** — все ограничения полей истории консолидированы в единую константу `HISTORY_FIELD_MAX_LENGTH` вместо разбросанных захардкоженных лимитов.
- **Документация ROX ONE** — добавлен раздел Collections в руководство с акцентом на формат вложенных элементов title + properties.

## Исправления

- **Аутентификация MiniMax CN** — удалён некорректный алиас `minimax-cn → minimax`, добавлен лёгкий прямой HTTP-тест для Pi-провайдеров и удалён префикс MiniMax для совместимости с CN API.
- **Инлайн-код в сообщениях** — текст между инлайн-бейджами (источники, навыки, файлы) теперь рендерится через Markdown-компонент, восстанавливая инлайн-код, жирный шрифт, курсив и ссылки.
- **Сквозная передача Zod/JSON Schema** — схемы Zod по умолчанию используют `.passthrough()` для соответствия семантике JSON Schema, где `additionalProperties` по умолчанию равно `true`. Также сохранён `additionalProperties` при round-trip схемы инструментов MCP-прокси. Это устраняет тихую потерю полей в инструментах со слабо типизированными схемами.
- **Самоподписанные TLS-сертификаты** — принятие самоподписанных сертификатов для сконфигурированного источника удалённого сервера, устраняющее ошибку `ERR_CERT_AUTHORITY_INVALID` на `wss://`-соединениях.
- **Вложение файлов в режиме тонкого клиента** — кнопка скрепки теперь использует браузерный `FileReader` API вместо серверного `fs.readFile()`, устраняя тихие ошибки при отличающихся файловых системах клиента и сервера.
- **Преобразование URL в ссылки** — удаление замыкающих markdown-символов (`**` и т.д.) из преобразованных в ссылки URL, которые создавали битые ссылки.
- **Бейджи действий автоматизаций** — боковая панель теперь показывает корректные бейджи «Prompt» / «Webhook» на основе фактических типов действий вместо захардкоженного «Prompt».
- **Резервный путь упакованного сервера** — добавлен резервный вариант `dist/resources` для сборок, где компоновка вывода `extraResources` отличается.
- **Отсутствующие метки в `$ROX_EVENT_DATA`** — все payload-ы событий автоматизаций теперь включают текущий массив `labels` сессии, поэтому вебхуки и скрипты имеют доступ к данным меток.
- **Мультивыбор несмежных сессий** — Cmd-click теперь всегда переключает выделение (стандартное поведение ОС); открытие в новой панели перемещено на Cmd-Shift-click.
- **Автодополнение @-упоминаний с пробелами** — пробелы теперь допускаются в поисковых запросах упоминаний файлов (например, `@app availability.md`). Меню автоматически закрывается в стиле Slack, когда пробел не даёт совпадений.
