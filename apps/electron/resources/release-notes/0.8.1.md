# v0.8.1 — Восстановление удалённых подключений, Docker Compose и исправления стабильности

## Возможности
- **Поток восстановления удалённого рабочего пространства** — Отключённые удалённые рабочие пространства теперь можно переподключить напрямую из переключателя рабочих пространств. UI переподключения предзаполняет текущий URL, позволяет обновить URL или токен, проверить подключение и сохранить без удаления и повторного создания рабочего пространства.
- **Развертывание сервера на базе Docker** — Добавлен `docker-compose.yml` для упрощения headless-развёртывания, Compose по умолчанию использует образ сервера GHCR, а ресурсы Web UI встроены в Docker-образ, так что один сервис обеспечивает и headless-, и браузерный доступ.
- **Миниатюры файлов в Web UI** — Браузерные клиенты теперь загружают миниатюры изображений через fallback `readFileDataUrl` вместо зависимости от Electron-специфичных `thumbnail://` URL.

## Улучшения
- **Единообразная сортировка меток** — Выборщики меток теперь используют одинаковую алфавитную сортировку в боковой панели, автодополнении `#`, контекстных меню правого клика, меню пакетных меток и выпадающих списках фильтров.
- **Ясность источников навыков** — Навыки уровня проекта теперь корректно разрешаются в SkillInfoPage, показывают свой источник понятнее, и навыки, управляемые проектом, больше не предлагаются для удаления из приложения.
- **Единообразие именования артефактов** — Скрипты сборки, CI-workflow, установочные скрипты, документация и ссылки для загрузки теперь единообразно используют множественное именование `ROX-ONE-*`.
- **Очистка при передаче удалённых сессий** — Сессии, переданные в удалённое рабочее пространство, теперь отбрасывают устаревшие пути working-directory, не существующие на целевом сервере.

## Исправления
- **Стабильность удалённых рабочих пространств** — Исправлено несколько режимов отказа удалённых рабочих пространств: приложение больше не зависает на splash screen при недоступном удалённом сервере, уменьшено мерцание баннера переподключения, переподключение ожидает реального соединения, а вызовы Claude OAuth/настройки маршрутизируются на сервер удалённого рабочего пространства при необходимости.
- **Обработка ошибок загрузки сессий** — Ошибки загрузки сессий, не связанные с транспортом, теперь отображаются явно вместо маскировки под пустое состояние рабочего пространства.
- **Обработка изображений и вложений** — Большие вложения теперь используют поблочное кодирование base64, проверки размеров изображений нефатальны при отсутствии `sharp`, а невалидные изображения отличимы от отсутствующей поддержки обработки.
- **Устойчивость подпроцесса Pi** — Повреждённые stdout-каналы больше не вызывают каскадных дублирующихся ошибок, а дедупликация ошибок подпроцесса корректно сбрасывается на границах жизненного цикла.
- **Совместимость Web UI / headless** — Web UI теперь разрешает рабочее пространство по умолчанию до рукопожатия WebSocket и избегает вызова desktop-only RPC-каналов при подключении к headless-серверу.
- **Значения по умолчанию для Docker** — Исправлены значения по умолчанию в Dockerfile и Compose для владения директорией `.rox`, портов и поведения томов для собственных серверных установок.

## Критические изменения
