# v0.7.6 — Кастомные эндпоинты и заголовки MCP

---

## Возможности

- **Поддержка кастомных заголовков MCP** — источники MCP с HTTP-транспортом теперь могут отправлять кастомные заголовки во всех запросах (обнаружение и вызов инструментов). Значения заголовков безопасно хранятся в хранилище учётных данных через `headerNames`.
- **Флаг `--workspace-dir` для validate-server** — CLI-флаг `--validate-server` теперь принимает `--workspace-dir` для тестирования предкоммиченных источников на конкретную структуру рабочего пространства.

## Исправления

- **Кастомный эндпоинт не сохранялся** — поле `customEndpoint` молча удалялось при сохранении конфигурации LLM-подключения, поэтому кастомные Anthropic-совместимые эндпоинты откатывались к API по умолчанию при перезапуске.
- **Неверные ID моделей для кастомных эндпоинтов** — значения по умолчанию для кастомных эндпоинтов теперь используют корректные ID моделей Anthropic вместо устаревших значений.
- **Цикл обновления OAuth для источников с браузерной авторизацией** — источники, требующие браузерной авторизации, больше не вызывают тихие попытки обновления OAuth, которые завершались ошибкой и зацикливались.
- **Захардкоженный «Claude Max» в ошибке** — сообщение об истечении сессии больше не упоминает «Claude Max» по имени.

## CI / Документация

- **Паритет OSS-сборки** — добавлены скрипты платформенной упаковки (`build-dmg.sh`, `build-linux.sh`, `build-win.ps1`) и macOS entitlements plist в OSS sync allowlist, что позволяет участникам сообщества собирать production-пакеты.
