# v0.8.7 — Локализация, исправления Bedrock и обновление API-токенов

## Возможности
- **Венгерский, немецкий и польский переводы** — Три новых языка с полным покрытием UI-строк в shell, всплывающем окне информации о сессии, секции файлов, компонентах rich text input и island. Польский включает корректную поддержку форм множественного числа.
- **Эндпоинт обновления API-токенов** — API-источники теперь могут указывать опциональный `renewEndpoint` для автоматического обновления токенов без полного OAuth. Полезно для API, выдающих кратковременные токены с выделенным механизмом обновления.

## Улучшения
- **Поддержка raw body для API-источников** — Прокси API-источников теперь корректно обрабатывает параметры `_rawBody` и `_contentType`, позволяя источникам отправлять не-JSON payload (например, form-encoded, XML).

## Исправления
- **Модели Bedrock по умолчанию и привязка к региону** — Исправлен список моделей по умолчанию для подключений Bedrock и добавлено разрешение inference-профилей с учётом региона, что исправляет ошибки, при которых начальный выбор моделей завершался ошибкой «model identifier is invalid». Bare Claude ID моделей без необходимого префикса `us.` inference-профиля теперь автоматически отфильтровываются.
- **Прокрутка выпадающего списка моделей** — Подменю выпадающих списков (например, селектор моделей) теперь прокручиваются, когда список превышает высоту viewport.
- **Доступ к файлам рабочего пространства** — Файлы в рабочей директории рабочего пространства теперь открываются корректно, исправляя ошибки «cannot open code file».
- **Освобождение блокировки сервера при выходе** — Файл блокировки сервера теперь корректно освобождается при выходе из приложения с усиленным обнаружением устаревших блокировок для предотвращения ложных срабатываний «another instance running» при восстановлении после краша.
- **Обновление сессий при устаревшем переподключении** — Перенос обновления сессий при устаревшем переподключении в транзакционное атомарное действие, предотвращающее состояния гонки, которые могли привести к потере сообщений после переподключения после спящего режима/пробуждения.

## Критические изменения
- Нет.
