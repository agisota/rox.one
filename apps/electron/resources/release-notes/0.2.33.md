## Возможности

### Бейджи состояния рабочего процесса
- Сессии теперь отображают своё состояние рабочего процесса в виде бейджа рядом с метками
- Нажмите для изменения состояния через выпадающее меню
- Выбор состояния доступен в меню автодополнения `#`
- Состояние сохраняется при переключении между сессиями

### Встроенный просмотрщик Diff
- Блоки кода с diff в markdown теперь отображаются с расширенным визуальным сравнением side-by-side
- Автоматический откат к обычному блоку кода при ошибке рендеринга

### Сохранение состояния раскрытия TurnCard
- Развёрнутое/свёрнутое состояние карточек ходов сохраняется при переключении между сессиями
- Состояние хранится в localStorage с ограничениями LRU (максимум 100 сессий)

## Улучшения

### Унифицированное управление версиями
- Консолидировано управление версиями: `package.json` теперь единственный источник истины
- Удалён `app-version.ts` в пользу прямого чтения `package.json`
- Скрипты сборки, CI-workflow и исполняемый код используют один и тот же источник версий
- Упрощены инструменты релиза (удалён `sync-version.ts`)

### Обработка состояния окна
- Улучшена обработка URL для восстановления состояния окна
- Удалена ненужная санитизация URL (параметры запроса извлекаются и пересобираются безопасно)

### Настройка SDK на Windows
- Улучшенное обнаружение ошибок и понятные инструкции для отсутствующей директории `.claude/skills`

## Исправления

- Исправлено несовпадение версий между `app-version.ts` и `package.json`, приводившее к некорректным манифестам релиза
- Состояние карточек ходов перенесено из глобальных атомов Jotai в привязанный к сессии localStorage для лучшей изоляции
