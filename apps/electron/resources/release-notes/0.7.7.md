# v0.7.7 — Уровни размышления, условия автоматизаций и исправления для Windows

---

## Возможности

- **5-уровневая система размышления** — новые настройки размышления: Выкл / Низкий / Средний / Высокий / Макс. Модели Anthropic теперь используют адаптивное размышление вместо устаревшего бюджетного подхода. Haiku использует корректные токенные бюджеты (6000/8000 вместо 31999). Генерация заголовков больше не расходует токены на расширенные рассуждения.
- **Условия автоматизаций** — новая система условий в стиле Home Assistant с поддержкой временных, основанных на состоянии и логических (и/или/не) композиций как опционального шлюза между триггером и действиями. Отображается как читаемый блок «If» в детальном просмотре автоматизации с описаниями на естественном языке (например, «имеет метку X», «режим разрешений изменён с explore на allow-all»).
- **YouTube и Google Search Console** добавлены как сервисы Google OAuth.
- **Параллельное выполнение `call_llm`** — вызовы `call_llm` теперь выполняются параллельно на бэкендах Claude и Pi вместо последовательного выполнения.
- **Серверный просмотр директорий** — файловый браузер теперь работает для удалённых/тонких клиентских подключений, исправлено выделение выбора папок.

## Исправления

- **Утечка метаданных в SDK** — поля `_intent`/`_displayName`, внедряемые для API, теперь корректно удаляются до попадания в SDK, включая regex-резервное удаление при ошибке парсинга JSON и удаление для не-SSE JSON-ответов. Это вызывало появление лишних параметров у модели и ошибки вызовов инструментов.
- **Крах MCP OAuth на Figma** — серверы вроде Figma, которые предоставляют `registration_endpoint`, но возвращают 403 для неутверждённых клиентов, больше не вызывают крах потока OAuth. Используется резервный client ID по умолчанию, а в имени MCP OAuth-клиента теперь используется префикс «Claude Code» для совместимости с Figma.
- **Окно OAuth-браузера на Windows** — исправлена семантика прерывания передачи аутентификации Claude, которая препятствовала открытию окна OAuth-браузера.
- **Цикл обновления MCP OAuth** — источники MCP, требующие авторизации через браузер, больше не застревают в состоянии `needs_auth` из-за тихих попыток обновления, возвращавших «Token refreshed» без отображения окна входа.
- **Отсутствующий обработчик `settings:getNetworkProxy`** — настройки сетевого прокси теперь можно сохранять. Удалено около 230 строк мёртвого кода из старого расположения обработчика.
- **Общие папки проектов между рабочими пространствами** — история рабочих директорий теперь привязана к каждому рабочему пространству.
- **Разрешение моделей для кастомных провайдеров** — исправлено разрешение моделей и выбор модели суммаризации SDK для кастомных провайдеров.
- **Состояние гонки WebSocket-клиента** — исправлено состояние гонки подключения и добавлено логирование подключения/отключения.
- **Устаревшая привязка браузера** — исправлена привязка сессии к браузеру после уничтожения окна; методы браузерной панели теперь проверяют уничтоженные окна.
- **Проверка истории вебхуков** — проверка устойчива к граничным случаям.
- **Сетевой прокси не применялся к API-вызовам** — настройки прокси, сконфигурированные в настройках, применялись только к основному процессу Electron, но не к подпроцессам SDK, выполняющим фактические API-вызовы. Переменные окружения прокси теперь внедряются во все подпроцессы, а сетевой перехватчик маршрутизирует запросы через сконфигурированный прокси с поддержкой обхода `NO_PROXY`.
- **Переменная окружения `ROX_DEBUG`** — больше не игнорируется автономным сервером.

## Улучшения

- **Унифицированное сопоставление автоматизаций** — рефакторинг логики сопоставления автоматизаций и валидации условий для обеспечения согласованности и корректности.
- **Удалена избыточная блок-схема** — удалена визуализация Mermaid из информационной панели автоматизаций; блоки When/If/Then описывают автоматизации более наглядно.
