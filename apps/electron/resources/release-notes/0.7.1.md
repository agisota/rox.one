# v0.7.1 — Стабильность и надёжность

---

## Улучшения

- **Движок Pi agent 0.56.2** — обновлён с 0.55.0 с поддержкой GPT-5.4 (openai, openai-codex, azure-openai-responses), fallback gpt-5.3-codex для GitHub Copilot, интеграцией нативных бесед Mistral и поддержкой провайдера OpenCode Go
- **Заголовки сессий автоматизаций** — сессии, инициированные автоматизациями, теперь используют имя автоматизации как заголовок сессии вместо усечённого фрагмента промпта; генерация заголовка ИИ пропускается для таких сессий (861a5974, 79b9ebb4)
- **Флаги CLI validate-server** — добавлены флаги `--disable-spinner` / `--no-spinner` для CI-окружений; validate-server автоматически загружает временное рабочее пространство и LLM-подключение при их отсутствии (84b3378c, 1ea478d7)

## Исправления

- **Надёжность создания веток** — исправлено состояние гонки, при котором `ERR_STREAM_WRITE_AFTER_END` от `ProcessTransport` SDK оставлял создание ветки в состоянии бесконечной загрузки; добавлен 15-секундный таймаут с отменой по прерыванию для зависших предзапросов (8ff37102, 0dc1d2e6)
- **Ложные срабатывания Base64** — эвристическое обнаружение base64 заменено строгим конвейером канонизации (регулярное выражение charset, нормализация, автодополнение, проверка кругового обхода) для устранения ложных срабатываний на английском тексте и другом контенте, не являющемся base64
- **Восстановление сессий между машинами** — сохраняется очищенный `sdkSessionId` при восстановлении с истёкшей сессией, предотвращая повторные сбои из-за устаревших идентификаторов при синхронизации сессий между машинами
- **ENOENT при spawn во время автообновления** — обнаружена подмена app bundle во время автообновления с использованием структурированных полей ошибки + регексного fallback с ограниченной однократной попыткой повтора
- **Дублирование LLM-подключений при повторной авторизации** — исправлено устаревшее замыкание React в `handleReauthenticateConnection`, которое генерировало новые slug (например `chatgpt-plus-2`) вместо переиспользования существующих; добавлена защита `updateOnly` на стороне сервера (b735f9df)
- **Истечение токена Codex** — ошибки истечения авторизации Pi/Codex теперь проходят через конвейер `typed_error` auth-retry вместо отображения в виде красных сообщений об ошибках; глобальный mutex обновления предотвращает гонки обновления токенов между сессиями (1d85bc87)
- **Зависание «Проверить сейчас» в настройках на 0%** — унифицирована политика `autoDownload` между проверкой в настройках и нативным меню, чтобы события прогресса загрузки приходили корректно (b8ad8c44)
