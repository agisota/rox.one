# v0.3.2 — Режим фокуса и улучшения OAuth

Оставайтесь сосредоточенными с новым режимом фокуса. Потоки OAuth стали надёжнее и безопаснее.

---

## Режим фокуса

Новый режим без отвлекающих элементов, помогающий сосредоточиться на работе.

### Как это работает

**Переключение фокуса**
Скройте боковую панель и максимизируйте рабочее пространство, чтобы оставаться сфокусированным на текущей задаче.

---

## Поддержка паролей Basic Auth

Источники с базовой аутентификацией теперь могут дополнительно требовать поле пароля.

### Что нового

**Гибкая аутентификация**
Настройте источники basic auth с `passwordRequired: true`, чтобы отображались поля имени пользователя и пароля в диалоге учётных данных. API-интерфейсы вроде Ashby, использующие API-ключ как имя пользователя с пустым паролем, теперь поддерживаются.

## Улучшения

### Тема и UI
- Улучшения и полировка темы
- Улучшения тёмного режима — включает исправление кэша иконок: настроенные иконки теперь имеют приоритет над автоматически обнаруженными локальными файлами
- Улучшения переключения рабочих пространств
- Централизованная структура меню

### OAuth и безопасность
- Прогрессивное обнаружение метаданных OAuth по RFC 8414 — исправляет OAuth с Ahrefs MCP и аналогичными серверами, имеющими несколько сегментов пути в URL
- Удалены встроенные учётные данные Google OAuth из сборки — пользователи теперь предоставляют свои учётные данные через конфигурацию источника
- Добавлена проверка валидности учётных данных в `source_google_oauth_trigger`

### Документация
- Консолидирована документация по настройке Google OAuth на одной странице
- Добавлена документация параметра `passwordRequired` для базовой аутентификации

## Безопасность

- **Исправлена уязвимость path traversal** в IPC-обработчике `STORE_ATTACHMENT` — вредоносные значения sessionId могли ранее выйти за пределы директории рабочего пространства и записать файлы в любое место файловой системы
