# v0.4.4 — Переработка аутентификации Codex и исправления стабильности

Комплексные исправления аутентификации источников Codex, обновления OAuth-токенов и стабильности сессий. Этот релиз устраняет постоянные сбои аутентификации, краши из-за нехватки памяти и гонки переподключений.

---

## 🔐 MCP и обновление OAuth-токенов

- Использована взаимоблокировка обновления OAuth-токенов, при которой источники с истекшими токенами навсегда оставались в состоянии `needs_auth` до ручной переаутентификации (dc019c66)
- Исправлено: `isOAuthSource()` смешивал проверку типа с проверкой состояния аутентификации, из-за чего менеджер обновления токенов никогда не обновлял истекшие токены (dc019c66)
- Исправлено несоответствие `ensureFreshToken`/`needsRefresh`, при котором отсутствие `expiresAt` вызывало бесконечный цикл обновления-и-пропуска каждый ход (7a0f09e1)
- Проактивное обновление токенов, когда `expiresAt` не определен, с обработкой предварительно существующих учетных данных, сохраненных без срока действия (874edf28)
- Значение `expires_in` по умолчанию 3600с, если оно отсутствует в ответах OAuth-токенов, предотвращая ситуации, когда токены никогда не обновляются (874edf28)
- Восстановление состояния аутентификации (диск + в памяти) после успешного обновления токена (dc019c66)

## 🔌 Codex

- Исправлены заголовки аутентификации MCP: генератор конфигурации теперь записывает `http_headers` вместо `headers`, что ранее незаметно отбрасывало токены аутентификации для всех MCP-источников (874edf28)
- Источники, аутентифицированные в середине сессии, теперь автоматически активируются и немедленно доступны для Codex (b8d358eb)
- Исправлена гонка между отложенным переподключением и следующим ходом Codex после OAuth в середине сессии (d190b9b1)
- Усилены гонки переподключений и накопление вывода для предотвращения взаимоблокировок и потерянных промисов (7a0f09e1)
- Улучшено обнаружение инструментов Codex и руководство при сбоях аутентификации (a5158d66)
- Предотвращены краши OOM от неограниченного вывода инструментов — вывод команд и MCP усекается на 100K символов в адаптере событий, предохранительный лимит 200K символов на уровне слоя сохранения (f66241d6)

## 🤖 Copilot

- Исправлено: CLI Copilot не найден в упакованных сборках — нативный бинарник теперь поставляется для каждой платформы, что устраняет ошибки `import_meta.resolve is not a function` при входе (eb02eeb9)
- Добавлен 30-секундный таймаут для получения модели аутентификации Copilot для предотвращения бесконечного зависания (c8f46f59)
- Исправлено отсутствие перехватчика Copilot в конвейере сборки Windows (fc619f63)
